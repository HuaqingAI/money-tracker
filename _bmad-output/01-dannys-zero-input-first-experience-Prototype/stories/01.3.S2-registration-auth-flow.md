## What to Build

### Supabase 手机号 OTP 流程

```ts
// 发送验证码
const sendOtp = async (phone: string) => {
  setLoading(true)
  const { error } = await supabase.auth.signInWithOtp({
    phone: `+86${phone}`,
  })
  if (error) {
    showToast({ type: 'error', message: error.message })
  } else {
    setOtpSent(true)
    startCountdown(60) // 60秒倒计时
  }
  setLoading(false)
}

// 验证 OTP
const verifyOtp = async (phone: string, token: string) => {
  if (!agreed) {
    showToast({ type: 'warning', message: '请先阅读并同意用户协议' })
    return
  }
  setLoading(true)
  const { data, error } = await supabase.auth.verifyOtp({
    phone: `+86${phone}`,
    token,
    type: 'sms',
  })
  if (error) {
    showToast({ type: 'error', message: '验证码错误，请重试' })
  } else {
    const isNewUser = data.user?.created_at === data.user?.last_sign_in_at
    if (isNewUser) {
      router.replace('/(auth)/permission')
    } else {
      router.replace('/(app)/(tabs)/dashboard')
    }
  }
  setLoading(false)
}
```

### 验证码倒计时

```ts
const [countdown, setCountdown] = useState(0)

const startCountdown = (seconds: number) => {
  setCountdown(seconds)
  const timer = setInterval(() => {
    setCountdown(prev => {
      if (prev <= 1) { clearInterval(timer); return 0 }
      return prev - 1
    })
  }, 1000)
}

// 获取验证码按钮状态
// - loading: "发送中…"
// - countdown > 0: "重新获取 ({countdown}s)"，禁用
// - countdown === 0: "获取验证码"，启用
```

### 页面状态机

| 状态 | 触发 | UI 变化 |
|------|------|---------|
| `idle` | 初始 | 显示手机号输入 + "获取验证码"按钮 |
| `otp-sending` | 点击"获取验证码" | 按钮显示 loading spinner |
| `otp-sent` | 发送成功 | 显示验证码输入框 + 倒计时 |
| `otp-verifying` | 点击"登录/注册" | 提交按钮显示 loading |
| `error` | 验证失败 | Toast 错误提示，不清除输入 |
| `success` | 验证成功 | 路由跳转 |

## Acceptance Criteria

### 功能验证

| # | 项目 | 预期 |
|---|------|------|
| 1 | 手机号格式校验 | 非法格式时"获取验证码"禁用 |
| 2 | 发送验证码 | 成功后验证码输入框展开，倒计时 60s |
| 3 | 倒计时期间按钮禁用 | 不可重复发送 |
| 4 | 协议未勾选提交 | Toast 警告"请先阅读并同意用户协议" |
| 5 | OTP 验证成功（新用户） | 跳转 01.4 Permission |
| 6 | OTP 验证成功（老用户） | 跳转 01.7 Dashboard |
| 7 | OTP 错误 | Toast 错误提示，保留输入 |
| 8 | 加载期间防重复提交 | 按钮 loading 状态，不可重复点击 |

### 安全要求

- 手机号格式在客户端验证（防止无效请求）
- OTP 超时（Supabase 默认 60s）后提示重新获取
- 不在客户端存储明文验证码
