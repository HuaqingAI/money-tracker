## Layer 2: Project Context (Initial Load)

### From Product Brief
**Business:** Money Tracker, AI-driven zero-input financial butler for Chinese market
**Challenge:** Users hate manual bookkeeping but need financial clarity
**Goal:** Eliminate manual bookkeeping through AI auto-capture and intelligent management

### User Archetypes (to deepen)
1. **大强 (Danny)** - Married, 30s, mortgage, family financial clarity issues
2. **小丽 (Lily)** - Young urban worker, 25, paycheck-to-paycheck, small purchases add up
3. **小美 (Mei)** - Side-hustle worker, 28, needs multi-identity accounting
4. **人情用户** - Active social life, gift/reciprocity network management

### From Brainstorming (33 ideas)
- Key themes: Zero-input accounting, multi-identity, AI butler, gift accounting, future prediction
- Breakthrough: Gift accounting as blue ocean differentiator
- Positioning shift: From "bookkeeping tool" to "AI financial butler"
- Interaction model: Push notifications + AI conversation (dual mode)

### From Technical Research
- Android: NotificationListenerService captures 80-90% transactions automatically
- iOS: Siri voice (3s) + Widget (1s) + Screenshot OCR as alternatives
- CSV import: Alipay (GBK) + WeChat (UTF-8) as universal fallback
- Architecture: Local-First + Next.js API Routes + Supabase

### Constraints
- Technical: React Native (Expo) + Next.js + Self-hosted Supabase + DeepSeek/Qwen + Alibaba Cloud ECS
- Business: Sue solo developer, no hard deadline, MVP budget ~2000-4000 RMB/year
- Model: B2C Freemium (F1-F3 free, F4-F13 paid)

### Strategic Direction
- Primary: Validate zero-input experience (MVP North Star: AI capture coverage %)
- Secondary: Differentiate via gift accounting + multi-identity
- Future: AI butler intelligence layer
