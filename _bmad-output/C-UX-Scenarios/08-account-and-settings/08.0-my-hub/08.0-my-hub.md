## Overview

**Page Purpose:** 底部 Tab "我的"的首页，作为用户的个人空间入口。提供身份识别（Profile Card）、P2 功能的统一入口（人情账、多身份、分类管理等）、以及通往设置和帮助的路径。核心使命是让散落在各场景中的功能有一个"回家"的地方。

**User Situation:** 大强点击底部 Tab "我的"，心态是"看看我的账户状态"或"我想找某个功能但不确定在哪"。这不是一个用户会长时间停留的页面，而是一个快速分发的枢纽。

**Success Criteria:**
- 用户能在 3 秒内找到目标功能入口
- Hub 页到 08.1 Settings 的转化路径 <= 1 次点击
- Hub 页功能入口的点击分布覆盖所有 P2 功能（无死角入口）
- 会员状态展示后，付费页面的点击率 >= 5%

**Entry Points:**
- 底部 Tab "我的"（主要入口）
- 深度链接

**Exit Points:**
- 点击 Profile Card -> 08.2 Profile / Account
- 点击功能卡片 -> 04.1 / 05.1 / 02.2
- 点击"记一笔" -> 06.1 Manual Entry
- 点击工具列表项 -> 01.5 / 06.2
- 点击"设置" -> 08.1 Settings
- 点击"帮助与反馈" -> 帮助页（非本规格范围）
- 底部 Tab 切换到其他页面

## Design Decisions

1. **Hub 而非直接 Settings** — "我的" Tab 应该是"我的空间"而非"系统设置"。竞品惯例（微信/支付宝/微博）都用 Hub。08.1 Settings 太重（781行，7 个 Section），直接当首页会让人觉得在"配置系统"而非"管理我的东西"
2. **Profile Card 从 08.1 提升到 Hub** — 头像+昵称+会员状态是用户身份的第一视觉锚点，应该在 Tab 首页就看到，而非进入设置后才看到
3. **1x3 核心功能卡片而非 2x3 网格** — 2x3 网格信息密度过高，视觉上像工具面板而非精品个人空间。只保留三个核心付费功能（人情账/家庭账本/多身份），留白充足，层次清晰
4. **手动记账独立卡片，介于功能卡片和列表之间** — 手动记账是数据来源的重要兜底，不能藏进列表项。但它不是核心付费功能，不该跟 1x3 抢位置。独立卡片 + "AI 没捕到的，手动补上"副文案，既突出入口又强化产品叙事。Dashboard 已有 FAB "+" 按钮作为首页入口，Hub 的卡片是第二入口
5. **工具类入口降级为列表项** — 导入数据（一次性操作）和分类管理（低频）降级到列表，与设置/帮助同一视觉层级，符合使用频率
6. **付费功能显示但锁定** — 免费用户能看到所有功能入口（带锁标），点击跳转订阅页。这比"隐藏付费功能"更好，因为用户知道"还有更多"，提升升级动机
7. **Hub 页保持轻量** — 不展示数据、不展示图表、不展示洞察。Hub 的职责是"分发"不是"展示"。数据展示留给 Dashboard(01.7)
8. **头像 64pt vs Settings 48pt** — Hub 是"个人空间"入口，头像大一号传达"这是你的地方"
9. **"设置"降级为菜单列表项** — Settings 从 Tab 首页降级为 Hub 的一个菜单入口，符合用户心智模型：设置是低频操作，不应占据 Tab 首页的核心位置
10. **会员 Badge 始终可见** — 无论 Free/Trial/Active/Expired 都展示，让用户随时感知会员状态。特别是 Trial 状态显示到期日期，制造紧迫感
