## Overview

**Page Purpose:** 作为用户的"控制中心"，提供数据安全透明度（正面回应 D5-）、通知偏好配置、权限管理、以及通往个人资料/账户的入口。核心使命是让大强确认"我的数据是安全的"，安心继续使用。

**User Situation:** 大强首次导入账单后，忽然想到：银行卡消费、支付宝交易、家庭收入全在这个 APP 里了——数据安全吗？他从 Dashboard 底部 Tab "我的"进入设置页，带着一种"确认安全后就能放心"的心态快速浏览。

**Success Criteria:**
- 用户能在 10 秒内找到数据安全说明
- "数据安全"Section 的展开阅读率 >= 30%
- 设置页访问后 7 天内卸载率显著低于未访问用户
- 零用户因找不到权限管理入口而联系客服

**Entry Points:**
- 从 08.0 My Hub 点击"设置"进入（主要入口）
- 从 01.4 Permission 页"可随时在设置中开启"引导回来
- 从系统推送/洞察推送点击"管理通知偏好"进入（深度链接）

**Exit Points:**
- 点击"个人资料" -> 08.2 Profile / Account
- 点击返回 / 底部 Tab -> Dashboard 或其他 Tab 页
- 点击"关于我们" -> 关于页（独立页面，非本规格范围）

## User Mental State

| 维度 | 状态 |
|------|------|
| 触发事件 | 导入账单后想起"这么多财务数据都给了这个 APP"，或者收到太多/太少推送通知 |
| 情绪基调 | 低频审视（"我确认一下安全就行"），不紧急但重要 |
| 希望 | 快速确认数据加密存储、不会泄露，顺便调一下通知频率 |
| 恐惧 | D5-："银行卡消费、微信转账、家庭收入全在里面，这个 APP 靠谱吗？" |
| 内心问题 | "数据存在哪？""加密了吗？""怎么删除我的数据？""怎么关掉烦人的推送？" |
| 耐心程度 | 中高 — 主动进来的，愿意花 30 秒阅读安全说明，但不想看长篇隐私政策 |

## Content & Language Strategy

**叙事弧线：** 设置页无需叙事弧线，但信息架构隐含优先级排序。

| 优先级 | Section | 心理作用 |
|--------|---------|---------|
| 1 | 数据与隐私 | "我的数据是安全的" -- 直接回应 D5- |
| 2 | 通知偏好 | "我能控制它打扰我的频率" -- 掌控感 |
| 3 | 通用设置 | "我能按自己的习惯配置" -- 个性化 |
| 4 | 关于/退出 | "需要的时候能找到" -- 安全网 |

**语气：** 简洁、功能性、透明。设置页不需要营销语言，每个条目的描述用最少的字说明白。

**数据安全说明语气：** 与 01.4 Permission 页的隐私承诺保持一致的诚恳、透明风格。用具体技术名词（AES-256、HTTPS/TLS）建立专业可信度，但用口语化的描述解释技术含义。

**禁用词：** "绝对安全"（没有绝对安全）、"请放心"（空洞）

**语言：** 界面简体中文，代码/后端英文

## On-Page Interactions

| 交互 | 触发 | 行为 | 反馈 |
|------|------|------|------|
| 页面加载 | 从 08.0 Hub 点击"设置"进入 | 加载设置项状态 | 内容淡入 |
| 点击返回 | 左上角返回 / 安卓返回键 | 返回 08.0 My Hub | 页面右滑 |

## Design Decisions

1. **数据安全置于第一组** -- 设置页的信息架构隐含产品价值观排序。把"数据与隐私"放在最显眼位置（紧接 Profile Card），无声传达"我们最看重你的数据安全"
2. **安全说明内嵌展开而非跳转** -- 减少页面跳转层级，让用户在设置页内即可完成安全确认。4 项承诺与 01.4 Permission 的 3 项呼应但升级（增加了传输加密和用户控制权）
3. **通知偏好按功能分类而非频率** -- 用户更容易理解"AI 洞察"和"周报"的区别，而不是"高频/低频"的抽象分类
4. **通知偏好 Disabled 联动系统权限** -- 系统通知权限未开启时，APP 内的偏好设置没有意义。置灰 + 提示引导用户先开启系统权限
5. **退出登录用红色文字而非红色按钮** -- 低视觉权重防止误触，但红色传达"破坏性操作"语义
6. ~~**Profile Card 整体可点击**~~ — Profile Card 已移至 08.0 My Hub，Settings 页不再包含 Profile Card
7. **深色模式三态设计** -- "关闭/开启/跟随系统"覆盖所有用户偏好，跟随系统为推荐默认值
8. **MVP 语言仅中英文** -- 产品面向中国市场，中文为默认，英文为备选。后续版本可扩展
9. **版本号放在页面最底部** -- 开发调试和客服排查需要，用户日常不关注
