## Overview

**Page Purpose:** 创建家庭共享账本并邀请小芳加入。这是 D1+（用共同数字讨论家庭开支）和 D5+（让老婆也能看到同一份数据）的实现载体。大强刚领取免费会员，内心带着"领了就用起来"的积极心态，需要尽快完成创建 + 邀请的闭环，让小芳也进来。

**User Situation:** 大强刚领取了免费会员体验，此时心态积极 -- 免费领取消除了金钱顾虑，但"领了会员不用等于白领"的心理驱动他积极完成后续步骤。D2-（不想再因为钱跟老婆吵）的正向解决希望是核心动力，同时存在"小芳不愿意用怎么办"的担忧。页面需要让创建流程极简，同时在邀请环节给大强信心："小芳收到这个邀请会愿意点。"

**Success Criteria:**
- 账本创建完成率 >= 95%（从进入到创建成功，几乎无流失）
- 邀请发送率 >= 85%（创建后发出邀请）
- 邀请接受率 >= 50%（小芳收到后 7 天内接受）
- 创建到发送邀请平均时长 < 30s

**Entry Points:**
- 从 02.1 Subscription 领取成功后点击"创建家庭账本"进入（主场景路径）
- 从 01.7 Dashboard 账本切换器选择"创建家庭账本"进入（已是会员但未创建家庭账本）
- 从 08.x 设置 > 家庭账本 > "创建"进入（通用管理入口）
- 从任何功能中"家庭账本"相关入口进入（如报表页"邀请家人一起看"提示）

**Exit Points:**
- 发送邀请后 -> 02.3 Beneficiary Tagging
- 点击"稍后邀请" -> 02.3 Beneficiary Tagging（跳过邀请）
- 左上角返回 -> 来源页面（放弃创建，已领取会员功能保留）

## User Mental State

| 维度 | 状态 |
|------|------|
| 触发事件 | 刚领取免费会员，从领取成功页/Dashboard/设置进入 |
| 情绪基调 | 行动力强 + 积极（"免费的，赶紧弄好让小芳也能看到"） |
| 希望 | "简单设置一下就能邀请小芳"、"小芳收到后会愿意加入" |
| 恐惧 | "设置太复杂我就烦了"、"小芳会不会觉得我监控她？" |

| 耐心程度 | 高 -- 免费无成本，好奇心 + 解决夫妻财务沟通痛点的动力驱动 |
| 关键信号 | 看到"各自隐私消费不共享"时会松一口气 |

## Content & Language Strategy

**核心叙事：从"我的"变成"我们的"**

整个页面的文案策略围绕"共同"和"一起" -- 不说"管理家庭财务"（功能语言），说"同一份事实"（情感语言）。大强来这里不是为了"创建一个数据库"，而是为了"让下次讨论花销时有据可说"。

| 状态 | 叙事框架 | 核心信息 |
|------|---------|---------|
| Create | 情感共鸣 + 隐私安心 | "同一份事实，各自隐私保护" -- 打消顾虑再行动 |
| Invite | 简单行动 + 信心传递 | "微信发一下就好" -- 让邀请感觉像发条消息 |
| Waiting | 积极引导下一步 | "先做受益人标注" -- 等待时间不浪费 |

**语气：** 温暖、轻快、鼓励。传达"这是好事"而非"这是任务"。

**禁用词：** "监控"、"管控"、"记录对方消费"（会触发小芳/大强的隐私警觉）

**语言：** 界面简体中文，代码/后端英文

**边界情况：**
- **已有家庭账本：** 如果用户重新进入（从设置或 Dashboard），直接显示 Invite State 或 Waiting State（根据是否已发送邀请判断），不再显示 Create State
- **邀请链接过期：** 邀请链接 7 天有效。过期后需要重新生成。Waiting State 的"再次发送"会自动生成新链接
- **被邀请人已有账号：** 小芳已是 APP 用户 -> 点击邀请链接直接加入家庭账本（无需重新注册）
- **被邀请人无APP：** 小芳未安装 -> 邀请链接引导先下载 APP -> 注册 -> 自动加入家庭账本
- **微信未安装：** 微信分享按钮灰化 + 提示"请先安装微信"，其他两种方式正常可用
- **邀请被拒绝：** 被邀请人主动拒绝 -> 大强的家庭账本仍可用（个人使用），可重新邀请同一人或他人
- **多次邀请不同人：** MVP 阶段家庭账本仅支持 2 人（创建者 + 1 位受邀者），第二次邀请会提示"当前方案最多 2 人"
- **创建后 APP 被杀死：** 账本已创建（服务端持久化），重新打开后进入 Invite State
- **弱网分享：** 微信分享依赖微信 SDK，弱网时微信 SDK 自身处理重试。复制链接不依赖网络
- **账本名称冲突：** 同一用户不可创建同名账本，提示"已有同名账本"

## Design Decisions

1. **"同一份事实"而非"创建共享账本"** -- 页面标题和开场用情感语言而非功能语言。大强不是在"创建数据库"，他是在为"不再各说各的"做准备。Trigger Map Key Insight：共享事实消解冲突
2. **隐私说明前置于创建按钮** -- 在大强点击"创建"之前就回答他的核心顾虑："我的隐私消费她能看到吗？"消除恐惧后再行动，转化率更高
3. **预填默认账本名** -- "我们的家庭账本"直接可用，绝大多数用户无需修改。最小化输入 = 最小摩擦 = 呼应 D3-（害怕手动输入）
4. **微信分享为主推** -- 在中国，"发给老婆"几乎 = "微信发给老婆"。微信按钮用微信绿做视觉主导，复制链接和二维码为补充方案
5. **二维码面对面场景** -- 夫妻面对面时，"你扫一下"比"我发你微信"更自然。二维码默认折叠，点击展开，不占用主流程视觉空间
6. **"稍后邀请"不说"跳过"** -- "跳过"暗示"这步不重要"，而实际上邀请是场景核心目标。"稍后邀请，先设置受益人"引导用户前进而非放弃
7. **Waiting State 不阻塞** -- 邀请发送后不强制等待小芳接受，而是引导去做受益人标注。等待时间被有意义的操作填充
8. **隐私卡片跟随状态变化** -- Create State 说"你的隐私受保护"（打消创建顾虑），Invite State 说"对方会看到什么"（给大强信心去邀请）。同一个组件位置，不同状态服务不同心理需求
9. **MVP 限制 2 人** -- 家庭账本 MVP 仅支持创建者 + 1 位受邀者。简化权限模型和数据同步复杂度。后续版本扩展为多人家庭

## Technical Notes

- **创建 API：** `POST /api/ledgers` body: `{ name, type: "family" }` -> 返回 `{ ledger_id, invite_link, invite_code }`
- **邀请链接格式：** `https://{domain}/invite/{invite_code}`，H5 落地页引导下载或直接打开 APP（Deep Link）
- **邀请链接有效期：** 7 天，过期后需重新生成（`POST /api/ledgers/{id}/invite/refresh`）
- **微信分享：** 使用微信 SDK `ShareMessageToWX`，分享类型为 Mini Program Card（如有小程序）或 Web Page
- **二维码生成：** 前端使用 `react-native-qrcode-svg` 生成，编码内容为邀请链接
- **账本状态持久化：** 账本创建后服务端持久化，邀请状态（pending/accepted/expired）通过 `GET /api/ledgers/{id}` 获取
- **成员上限：** MVP 阶段 `max_members = 2`，后端校验
- **数据共享机制：** 家庭账本不复制交易数据，而是通过"共享标记"让双方可见被标记为共享的交易。隐私交易（未标记）仅本人可见
- **Dashboard 账本切换器：** 家庭账本创建成功后，01.7 Dashboard 顶部出现账本切换器 `[个人 ▾]`（参考 01.7 Design Decision #13）
