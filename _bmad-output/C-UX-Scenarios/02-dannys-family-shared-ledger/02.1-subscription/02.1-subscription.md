## Overview

**Page Purpose:** 让大强看到他即将免费解锁的全部高级功能，以"共享家庭账本"为核心价值锚点，一键领取会员。早期拉新阶段，所有新注册用户均可免费领取限时体验会员（L2+L3全部功能），页面隐藏付费入口，纯粹展示价值、消除顾虑、驱动领取。

**运营策略（早期拉新）：**
- **免费领取：** 所有新注册用户可领取 {trial_days} 天免费会员体验
- **功能范围：** 全部付费功能（Layer 2 核心 + Layer 3 AI管家）
- **无需支付：** 页面不展示价格、不涉及支付流程
- **到期策略：** 免费体验到期后，付费转化页面（Paywall）由后续设计覆盖

**User Situation:** 大强用了两周免费版，已经认可"零记账"的核心价值（自动导入+分类+报表）。周末小芳问"这个月我们到底花了多少"，他想把报表直接分享给小芳但意识到更好的方式是让她也能随时看到。在 Dashboard 上看到"邀请家人共享"提示卡片，点击进入。

**Success Criteria:**
- 领取转化率 >= 60%（免费无门槛，应远高于付费转化）
- 页面平均停留时间 10-30秒（免费领取无需过多犹豫）
- 领取后进入 02.2 家庭账本设置的比例 >= 50%

**Entry Points:**
- 从 Dashboard "邀请家人共享"提示卡片点击进入（主要入口，场景触发）
- 从 设置 > 账户 > "领取会员"进入（通用入口）
- 从任何高级功能的锁定提示点击"免费解锁"进入（功能触发）

**Exit Points:**
- 领取成功 -> 02.2 Shared Family Ledger 共享家庭账本设置
- 点击关闭/返回 -> 返回来源页面（Dashboard / 设置 / 功能页）

## Content & Language Strategy

**语气：** 惊喜赠予感，像产品团队给早期用户的感谢礼物。"我们想让你先完整体验一下，觉得好再说"。

**核心策略：**
- **赠予而非推销：** "送你XX天"而非"免费试用"，语气从交易变为礼物
- **Context Hook 用场景驱动：** 从 Dashboard "邀请家人共享"卡片进入时，用户的心智模型已经是"我要让小芳也能看到"，功能列表中"共享家庭账本"排第一
- **Usage Stats 个人化：** 用用户自己的使用天数做情感连接——"你已经觉得好用了"
- **三重信任保障：** 无需绑卡 + 到期自动降级 + 隐私保护，逐一击破残余顾虑
- **功能展示而非推销：** 先列已有功能（强化价值认知），再列即将解锁功能（锦上添花）
- **Skip Link 正面表述：** "继续基础版"而非"放弃"

**禁用词：** "仅需"、"限时"、"错过"（避免营销焦虑感）、"记账"、"试用"（用"体验"代替，更正面）

**语言：** 界面简体中文，代码/后端英文

| 展开功能详情 | 点击会员功能行 | 展开该功能的详细说明面板 | Accordion 展开动画（smooth height transition） |
| 收起功能详情 | 再次点击已展开功能行 | 收起详细说明面板 | Accordion 收起动画 |
| 展开/收起 AI 管家 | 点击 AI 管家折叠入口 | 展开/收起 Layer 3 全部功能列表 | Smooth expand/collapse |
| 点击"免费领取" | 触摸主 CTA 按钮 | 调用后端 API 激活免费会员 | 按钮进入 loading 态（防止重复点击）-> 成功后显示 Success Overlay |
| 领取成功 | 后端返回 success | 显示 Claim Success Overlay | 庆祝动效 + 到期日期显示 |
| 领取失败 | 后端返回 error | 显示错误 Toast，按钮恢复可点击 | Toast 提示具体原因（网络异常/已领取过/活动结束） |
| 创建家庭账本 | 领取成功后点击主 CTA | 导航到 02.2 Shared Family Ledger | 页面左滑进入 |
| 稍后再设置 | 领取成功后点击次要链接 | 返回 Dashboard | 页面右滑退出 |
| 点击"先不了" | 触摸 Skip Link | 返回来源页面 | 页面右滑退出 |
| 返回 | 左上角返回 / 安卓返回键 | 返回来源页面 | 页面右滑 |

**动效原则：**
- 所有动画 < 0.6s，ease-out 曲线
- Gift Hero 入场动画为视觉引导，不阻塞交互（动画过程中按钮即可点击）
- 设备开启"减少动态效果"时跳过所有动画直接显示静态内容
- Success Overlay 庆祝动效同样尊重无障碍设置

**边界情况：**
- **已是会员（已领取过）：** 不应进入此页面。Dashboard 提示卡片和设置入口应替换为直接的功能入口（如"创建家庭账本"）。如意外进入，页面显示"你已是会员"状态卡片 + 到期日 + "管理会员"入口，隐藏领取按钮

- **免费领取活动结束：** 后端返回活动已结束错误，页面应优雅降级为付费订阅页面（未来设计）。当前 MVP 阶段可简单显示"活动已结束"Toast + 返回

- **网络异常：** 点击"免费领取"时检查网络——无网时 Toast 提示"请检查网络连接"，按钮恢复可点击
- **快速连续点击：** "免费领取"做防抖处理，loading 态期间 disabled
- **后端激活延迟：** loading 态超过 3 秒显示"正在为你开通..."文案，超过 10 秒显示"网络较慢，请稍候"

## Design Decisions

1. **赠予感而非试用感** -- "送你XX天"vs"免费试用XX天"，前者是礼物，后者是营销套路。用户情感完全不同
2. **隐藏付费，纯粹展示价值** -- 早期拉新阶段消除一切金钱相关信息，让用户零心理负担地领取并使用。付费转化留给到期前的 Paywall 页面
3. **无需绑卡是最强信任信号** -- "完全免费，无需绑定银行卡或支付方式"一句话消除"免费试用偷偷扣费"的行业负面印象
4. **到期自动降级而非自动续费** -- 零风险承诺，让用户放心领取。到期转化由独立的 Paywall 流程处理
5. **功能展示保持不变** -- 免费领取降低了行动门槛，但功能展示仍然重要——让用户知道自己"领到了什么"，增加使用动力
6. **Success Overlay 展示到期日** -- 明确期限管理预期，避免"突然发现会员没了"的负面体验
7. **多入口适配** -- Success Overlay 的 CTA 文案根据 entry_source 动态调整
8. **Skip Link 用"基础版"** -- 而非"免费版"，避免暗示当前版本"低级"
9. **AI 管家功能默认折叠** -- Layer 3 功能对场景（家庭共享）关联度较低，折叠处理降低信息负担
10. **活动结束降级策略** -- 当免费领取活动结束时，页面需能降级为付费模式（后续独立设计）

## Technical Notes

- **会员激活 API：** POST 请求，参数包含 user_id + campaign_id，返回 member_status + expiry_date
- **会员状态模型：** 后端维护统一会员状态（free_trial / active / expired / cancelled），支持未来向付费订阅平滑过渡
- **活动配置：** trial_days、campaign_active 等通过后端运营配置下发，支持动态调整（如30天→60天），无需发版
- **到期提醒：** 到期前 {reminder_days} 天推送通知（如到期前3天和到期当天），引导至 Paywall 页面
- **降级处理：** 会员到期后，已创建的家庭账本数据保留但功能锁定（可查看历史但不能新增/编辑），恢复会员后自动解锁
- **防刷策略：** 每个设备/账号仅可领取一次免费会员，后端做幂等校验
- **未来扩展：** 当免费领取活动结束时，此页面需替换为付费 Paywall（独立设计），建议使用 Feature Flag 控制页面模式（free_claim / paywall）
