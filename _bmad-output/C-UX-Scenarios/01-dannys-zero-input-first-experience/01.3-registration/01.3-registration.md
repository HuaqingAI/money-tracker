## Overview

**Page Purpose:** 以最低摩擦完成账户创建/登录，将 Onboarding 建立的期待转化为实际行动。微信一键登录和手机号验证码并列展示，"注册即登录"合一设计，消除新老用户的认知差异。

**User Situation:** 大强刚看完 Onboarding 三幕叙事（或直接跳过），期待值处于峰值。他已经被"AI 财务管家"的定位吸引，现在需要"走完最后一步"开始使用。但注册是**摩擦断点** — 他从"被吸引"变成"被要求付出"（交出个人信息）。核心矛盾：D3-（害怕手动输入）在注册页会被再次触发 — 注册本身就是"手动输入"。

**Success Criteria:**
- 注册/登录转化率 >= 75%（从进入本页到完成注册/登录）
- 微信登录占比 >= 60%（验证微信作为主要登录方式的假设）
- 平均完成时间 < 15秒（微信路径 < 5秒，手机号路径 < 15秒）
- 零用户在此页卸载（不能让注册变成"劝退页"）

**Entry Points:**
- 从 01.2 Onboarding Slide 3 点击"立即体验"进入
- 从 01.2 Onboarding 任意 Slide 点击"跳过"进入

**Exit Points:**
- 微信授权成功 -> 01.4 Permission 权限授权页（新用户）
- 微信授权成功 -> 01.7 Dashboard（已有账号，自动识别）
- 手机号验证成功 -> 01.4 Permission 权限授权页（新用户）
- 手机号验证成功 -> 01.7 Dashboard（已有账号，自动识别）

## Content & Language Strategy

**语气：** 轻快、安全、低压力。让注册感觉像"走个过场"而非"填写申请表"。

**关键策略：**
- "几秒搞定"设定时间预期，降低"又要填一堆"的恐惧
- "登录/注册"合一文案，新老用户无认知差异
- 微信提示"将获取您的微信昵称和头像"主动披露 = 主动建立信任
- 协议弹窗"同意并继续"一键完成 = 摩擦转移（从主流程中拿出来）

**禁用词：** "记账"、"立即注册"（避免压力词）

**语言：** 界面简体中文，代码/后端英文

**流转规则：**

| 方向 | 条件 | 目标 |

**边界情况：**
- **已有微信账号用户：** 微信授权后后端检测到 unionId 已存在，直接登录跳转 Dashboard，显示"欢迎回来"Toast
- **已有手机号账号用户：** 验证码通过后后端检测到手机号已存在，直接登录跳转 Dashboard
- **微信和手机号已绑定同一账号：** 无论哪种方式登录都进入同一账号
- **微信未安装：** 微信登录按钮灰化并显示提示"请先安装微信"，自动切换到手机号 Tab
- **快速连续点击登录按钮：** 防抖处理，loading 态期间按钮 disabled
- **验证码倒计时中切换 Tab 再切回来：** 倒计时继续，不重置
- **弱网/超时：** 15秒超时后显示网络错误 Toast，按钮恢复可点击
- **微信授权弹窗中用户切到后台：** 回到前台后正常处理回调结果
- **协议弹窗中点击查看协议全文：** 跳转协议页后返回，弹窗仍在，勾选状态保持

## Design Decisions

1. **微信和手机号并列 Tab 设计** — Sue 确认并列展示，而非主次分离。理由：两种方式都是国内主流，给用户明确选择权；微信 Tab 默认选中，通过默认行为引导多数用户走最短路径
2. **"注册即登录"合一策略** — 不区分注册和登录页面。无论微信还是手机号，后端自动判断新老用户，新用户自动创建账号，老用户自动登录。文案"登录/注册"传达这一概念
3. **协议勾选框 + 弹窗兜底** — Sue 确认使用 Checkbox 方案（合规稳妥），但优化为：未勾选时点击登录弹出"同意并继续"弹窗，一键同意后自动勾选并继续操作，避免打断感
4. **验证码6位自动提交** — 填满6位后自动提交验证，减少一次点击。保留显式"登录/注册"按钮作为兜底
5. **微信授权范围提前告知** — "将获取您的微信昵称和头像"主动披露授权范围，消除 D5-（数据安全）恐惧
6. **"几秒搞定"副标题** — 用具体时间承诺（"几秒"）将注册重新定义为"走个过场"，对抗 D3-（害怕手动输入）的心理压力
7. **产品名称占位符** — `{product_name}` 待品牌命名确定后全局替换

## Technical Notes

- **微信登录 SDK：** 使用微信开放平台移动应用 SDK，需要 AppID 和 Universal Links (iOS) / App Signature (Android) 配置
- **短信服务：** 需要接入短信验证码服务（阿里云短信/腾讯云短信），60秒倒计时 + 每日每号码上限5条
- **手机号格式：** MVP 仅支持中国大陆手机号（+86），国际号码后续版本扩展
- **安全：** 验证码5分钟有效，错误3次后锁定15分钟；微信 token 传输全程 HTTPS
- **本地缓存：** 注册成功后本地写入 auth token + user profile，后续启动直接进入 Dashboard
