## Overview

**Page Purpose:** "打开就是答案" -- 展示月度消费概览，让大强不需要做任何操作就能看到钱去哪了。这是产品核心承诺 D2+（不动手就知道钱去哪了）的兑现时刻。对于空状态用户，用引导卡片告诉他们如何获得数据，而不是展示一个冰冷的空白页。

**User Situation:** 大强的心态取决于他从哪条路径到达：
- **从 01.6 处理完成进入（黄金路径）：** 最佳状态。刚看完 AI 分类动画，内心已经被"AI 真的在分析我的消费"的信任感铺垫好了。现在他期待看到结果 -- "餐饮花了多少？交通呢？"。这是整个首次体验中**情绪最高点**，D2+ 的承诺正在兑现。
- **从 01.4 跳过权限进入：** 没有通知读取权限，也没有导入历史账单。大强是"先看看再说"的心态，对产品还在观望。他不会看到任何消费数据，但不能让他觉得"白下载了"。
- **从 01.5 跳过导入进入：** 可能有通知读取权限（Android 授权后跳过导入），也可能没有（iOS / 未授权）。有权限的用户可能有少量实时捕获的消费，无权限的用户状态同上。

**Success Criteria:**
- 有数据用户：首页停留时长 >= 30s（在主动浏览内容）
- 有数据用户：点击"查看完整报表"的转化率 >= 40%
- 空状态用户：点击引导卡片（开启权限 / 导入账单）的转化率 >= 35%
- 首次看到消费分类数据时的惊喜感（定性指标：用户测试中的"哇"时刻）

**Entry Points:**
- 从 01.6 Import Processing 处理完成自动跳转（有数据）
- 从 01.6 Import Processing 中途退出（用户按返回键 -> "去首页"，后台继续处理）
- 从 01.4 Permission "暂时跳过"进入（空状态）
- 从 01.5 Bill Import "跳过，先看看"进入（空状态或少量数据）
- 底部 Tab 导航切换（日常使用）
- APP 启动后直接进入（已登录用户的默认首页）

**Exit Points:**
- 点击"查看完整报表" -> 01.8 Monthly Report
- 点击分类卡片 -> 03.1 Transaction List（按分类筛选）
- 点击"+" -> 06.1 Manual Entry
- 点击"补充更多账单数据" -> 01.5 Bill Import（Full State 二次导入入口）
- 点击引导卡片"开启通知读取" -> 01.4 Permission（Empty State）
- 点击引导卡片"导入账单" -> 01.5 Bill Import（Empty State）
- 底部 Tab 切换到其他页面

## User Mental State

### Full State (从 01.6 处理完成进入)

| 维度 | 状态 |
|------|------|
| 触发事件 | AI 分类完成，"已识别 128 笔，覆盖率 97%"的庆祝动效刚结束 |
| 情绪基调 | 期待 + 惊喜（"让我看看钱到底花在哪了"） |
| 希望 | "马上看到结果，最好一眼就明白" |
| 恐惧 | 几乎没有 -- 信任已在前几步建立 |
| 内心问题 | "餐饮花了多少？""跟我感觉一样吗？""这个能发给小芳看吗？" |
| 耐心程度 | 高 -- 主动想看，愿意浏览 |
| 关键信号 | 看到具体金额数字时会与心理预期做对比（"3200？这么多？"） |

### Empty State (跳过权限/跳过导入进入)

| 维度 | 状态 |
|------|------|
| 触发事件 | 跳过了权限授权或账单导入，"先看看"心态 |
| 情绪基调 | 观望 + 轻微失望（"果然没数据"，但这是自己选的） |
| 希望 | "让我知道下一步该干什么，别让我觉得白下载了" |
| 恐惧 | "这个 APP 空荡荡的，是不是不适合我？" |
| 内心问题 | "我现在能干什么？""开始用了之后是什么效果？""要不先手动记一笔试试？" |
| 耐心程度 | 低 -- 没有沉没成本，随时可能流失 |
| 关键信号 | 引导卡片的文案能否唤起 D2+ 的欲望（"不动手就知道钱去哪了"） |

## Content & Language Strategy

**核心叙事：打开就是答案**

Dashboard 的文案策略是"极简数据叙事" -- 用数字说话，文字仅用于标签和引导。大强不需要被教育"记账很重要"，他需要的是"一眼看到结果"。

| 状态 | 叙事框架 | 核心信息 |
|------|---------|---------|
| Full State | 直接呈现结果 | "这个月花了 ¥12,847，餐饮最多" -- 数字即答案 |
| Empty State | 引导行动 | "不动手就知道钱去哪了，开启任一方式即可" -- D2+ 唤醒 |

**语气：**
- Full State：简洁、客观、无评判。是"镜子"而非"法官"（参考 Trigger Map Key Insight：真相镜子而非法官）
- Empty State：温暖、鼓励、低压力。不催促，给选择

**禁用词：** "你花太多了""应该节约""预算超支"（评判性语言违背"真相镜子"原则）

**语言：** 界面简体中文，代码/后端英文

**边界情况：**
- **从 01.5 跳过但有通知权限 (Android)：** 可能有实时捕获的少量消费。如果交易数 >= 1 则按 Full State 展示（即使只有几笔），否则按 Empty State + 提示"新消费到账后会自动出现"
- **月份无数据：** 月份切换到无数据月份时显示"本月暂无记录"轻量空状态（不展示引导卡片，因为不是首次体验）
- **只有支出无收入：** 正常展示支出和分类，收入和结余显示为"--"
- **分类不足 5 个：** 展示实际分类数，不补空位
- **单笔分类占比极高（>80%）：** 正常展示，不做特殊标注（如房贷占 50%+ 是正常的）
- **AI 覆盖率低（<70%）：** 正常展示覆盖率，不额外解释。后续版本可引导用户修正分类
- **下拉刷新时新增了交易：** 刷新后数字平滑过渡到新值，不跳变
- **从引导卡片去开启权限/导入后返回：** 如果成功获取了数据，Dashboard 自动切换到 Full State
- **Family Prompt 被关闭后：** 不再显示。如果用户后续想了解，可在设置中找到家庭共享入口
- **数据量极大（数百笔/月）：** Dashboard 仅展示汇总和 Top 5，不影响性能。详情在 Transaction List 中展示
- **从 01.6 中途退出，后台仍在处理：** 显示 Import Banner 实时展示进度。Banner 与 Empty/Full State 可叠加（如已有历史数据 + 正在导入新一批）
- **导入处理中 APP 被杀死后重新打开：** 检查后端是否有进行中的导入任务，如有则恢复 Import Banner；如果处理已完成但用户未看到完成通知，直接展示 Full State（含新数据）
- **二次导入的去重：** 后端处理，不在 Dashboard 层面体现。用户可能重复导入同一个 CSV，后端自动跳过已存在的交易
- **二次导入后数据量显著增加：** Summary 和 Categories 正常更新，数字使用动画过渡到新值

## Design Decisions

1. **"打开就是答案"而非"打开是仪表盘"** — Dashboard 不是一堆图表的集合，而是回答一个简单问题的地方："这个月钱花在哪了？"。总支出用最大字号，分类排行自上而下，一屏之内回答核心问题。D2+ 要求的是"不动手就知道"，Dashboard 就是那个"知道"的着陆点
2. **空状态引导卡片而非空白页** — 跳过权限和跳过导入的用户看到的不是冰冷的空列表，而是温暖的引导卡片。三个选项（通知权限 > 账单导入 > 手动记账）从"最佳"到"兜底"排列，给用户选择而非命令
3. **AI 覆盖率外露** — 把"AI 已自动识别 97% 的消费"直接展示给用户。这不是技术指标，而是信任指标 -- 用户看到这个数字就知道"真的不用手动记"。首次加载的动画填充让这个数字被"看到"而非被忽略
4. **真相镜子，不做评判** — 参考 Trigger Map Key Insight，Dashboard 只呈现事实（"餐饮 ¥3,200"），不说"你餐饮花太多了"。评判权留给用户自己。禁用任何评判性语言
5. **家庭共享提示轻触而非硬推** — 以可关闭的卡片形式提示"邀请家人一起看"，直接命中 D1+ 和 D5+，但不强迫。MVP 阶段点击后显示"即将上线"，为 Phase 2 埋种子
6. **月份切换支持手势** — 除了左右箭头，还支持整页左右滑动切换月份。大强可能想对比上个月的消费，手势操作更自然
7. **FAB 始终可见** — 即使在空状态，悬浮"+"按钮也始终可见。手动记账是 AI 未捕获消费的补充方案，随时可用
8. **Bottom Tab 四栏** — 首页、交易、AI、我的。四栏是移动端最通用的导航模式，每个入口对应一个核心功能区域
9. **Empty State 区分有/无通知权限** — Android 已授权但跳过导入的用户，不显示"开启通知"卡片（已开启），而是显示"新消费到账后会自动出现"的提示，让他们安心等待
10. **产品名称占位符** — `{app_name}` 待品牌命名确定后全局替换
11. **Import Banner 不回到 01.6** — 01.6 是一次性的"信任建立剧场"，用户中途退出后在 Dashboard 看到的是轻量级 Banner（进度 + 完成提示），不需要也不应该重新进入 01.6 的全屏沉浸式动画
12. **Full State 保留二次导入入口** — 不把导入操作"藏"在设置里。用户第一次导入了 3 个月，用了一段时间想补充更早的数据，在 Dashboard 上就能直接找到入口。卡片视觉权重低于 Report CTA，不抢核心内容的注意力
13. **Phase 2 多账本：切换而非汇总** — 家庭账本和多身份账本引入后，Dashboard 顶部增加账本切换器（下拉选择），每次只展示一个账本的数据视图。**不做跨账本汇总**，原因：(a) 家庭账本包含配偶消费，与个人账本简单相加会重复计算；(b) 小美场景中副业"支出"是经营成本，与个人消费混在一起没有决策价值；(c) 家庭账本本身就是两人共享维度的"汇总"，再叠加个人账本破坏了 D1+（同一份数据）的语义。MVP 阶段仅有个人账本，切换器不显示；Phase 2 加入家庭账本后出现切换器

## Technical Notes

- **数据聚合：** Dashboard 数据从后端 API 获取月度汇总（总支出、总收入、分类汇总），不在前端做大量计算
- **API 端点：** `GET /api/dashboard/monthly?month={YYYY-MM}&ledger={ledger_id}` 返回 `{ total_spending, total_income, categories: [{name, amount, percentage}], ai_coverage_rate }`。`ledger` 参数 MVP 阶段默认为个人账本，Phase 2 支持家庭账本等
- **缓存策略：** 当月数据缓存 5 分钟（因为可能有新交易），历史月份缓存 1 天
- **骨架屏：** 加载时展示骨架屏（Skeleton），总支出位置显示灰色块，分类列表显示 5 行灰色占位
- **金额格式化：** 使用 `Intl.NumberFormat('zh-CN')` 格式化，千分位分隔
- **月份切换：** 使用 React Native 的 PanGestureHandler 实现左右滑动，与点击箭头共用同一个月份状态
- **空状态检测：** 根据当月交易数 = 0 判断空状态。交易数 >= 1 即为 Full State
- **引导卡片条件：** 通知权限状态通过 NotificationListenerService 检测，平台通过 `Platform.OS` 判断
- **Family Prompt dismissed：** 使用 AsyncStorage 存储 `family_prompt_dismissed: true`
- **下拉刷新：** 使用 FlatList 或 ScrollView 的 RefreshControl
- **Import Banner 数据源：** 复用 01.6 的 WebSocket/SSE 连接获取实时进度。Dashboard 加载时检查 `GET /api/imports/active` 判断是否有进行中的任务
- **Import Banner 状态持久化：** 不做前端持久化。每次进入 Dashboard 都通过 API 检查后端状态，避免前端状态与后端不一致
