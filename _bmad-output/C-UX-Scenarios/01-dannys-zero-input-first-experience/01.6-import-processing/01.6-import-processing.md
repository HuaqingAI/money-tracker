## Overview

**Page Purpose:** 将 CSV 账单的 AI 分类处理过程可视化展示，让"等待"变成"信任建立"时刻。大强亲眼看到 AI 逐条识别交易并自动分类，从而验证"零记账"的核心承诺不是空话。三层目的：功能层（展示 AI 处理进度）、情感层（把被动等待变成主动观看）、信任层（亲眼看到分类过程 = 相信 AI 真的在工作）。

**Design Concept: 信任建立剧场** — 这不是一个被动的加载页。这是一场"信任建立剧场"——大强亲眼看到 AI 工作的时刻，把怀疑（"又是一个记账APP"）变成相信（"它真的在自动分类！"）。动效要张扬、有视觉冲击力。

**User Situation:** 大强刚在 01.5 Bill Import 页面成功上传了支付宝 CSV 账单文件，看到"上传成功，正在解析..."的提示后被自动跳转到这里。他已经完成了注册、授权、导出 CSV、上传等一系列步骤，沉没成本高，注意力高度集中——这是整个流程的"揭晓时刻"。

**Success Criteria:**
- 页面停留时间 5-15 秒（太短说明动画没看到，太长说明卡住了）
- 跳转完成率 >= 99%（自动跳转，不应有流失）
- 分类覆盖率展示 >= 90%（低于此值用户信心会打折）
- 后续 Dashboard 互动率 >= 80%（到达 Dashboard 后有操作行为）

**Entry Points:**
- 从 01.5 Bill Import CSV 上传成功后自动跳转（主要入口，唯一入口）
- 纯自动进入页面——用户不会主动导航到这里
- 入口时携带数据：upload_id（CSV 文件上传记录 ID）、预估交易笔数

**Exit Points:**
- 处理完成 -> 自动跳转 01.7 Dashboard（庆祝动画展示 2.5s 后）
- 用户按返回键 -> 弹窗确认后跳转 01.7 Dashboard（后端继续处理）
- 处理错误 -> "先看已识别的" -> 01.7 Dashboard（展示部分数据）
- 处理错误 -> "重试" -> 重新初始化动画

#### AI Brain Visual（AI 大脑视觉）

**OBJECT ID:** `processing-ai-brain`

| Property | Value |
|----------|-------|
| Type | Animated Illustration |
| Description | 中央一个抽象化的 AI 图形（脉冲环/神经网络节点），持续呼吸式脉动，表达"AI 正在思考" |
| Size | 120x120pt 居中 |
| Animation | 持续脉冲（scale 0.95-1.05, opacity 0.8-1.0），周期 1.5s |

#### Category Bubbles（分类气泡群）

**OBJECT ID:** `processing-category-bubbles`

| Property | Value |
|----------|-------|
| Type | Animated Bubble Group |
| Description | 围绕 AI 图形，分类气泡逐个出现并生长。每识别到新分类时新增一个气泡，已有分类的气泡随交易增多而变大 |
| Bubbles | 每个气泡 = 分类图标 + 分类名（如 餐饮、交通、购物） |
| Layout | 围绕中心 AI 图形环形分布，最多同时显示 6-8 个主要分类 |
| Animation (New) | fade-in + scale from 0 -> 1（300ms, spring easing） |
| Animation (Growth) | size 随该分类交易笔数线性增大（smooth resize） |
| Animation (Pulse) | 收到新交易时脉冲一次（scale 1.05->1.0） |
| Haptic | 新分类首次出现时轻微震动反馈（haptic, light） |
| Note | 气泡大小差异直观反映消费分布，大强一眼就能看到"哪个分类最大" |

#### Flying Labels（飞入标签）

**OBJECT ID:** `processing-flying-labels`

| Property | Value |
|----------|-------|
| Type | Animated Text Labels |
| Description | 交易摘要文字从底部飞入 AI 图形，短暂停留后化为对应分类气泡的一部分。张扬的视觉冲击 |
| Content | 交易摘要（如"美团外卖 ¥28.50"）间隔飞入 |
| Animation | translateY(+100) -> center（400ms）-> fade out into bubble（300ms） |
| Interval | 0.8-1.2s/条（正常），1.5-2s/条（少量交易），0.3-0.5s/条（大量交易抽样） |
| Note | 不需要展示每一条交易，间隔抽样展示即可，确保节奏感。动画循环播放直到后端完成 |

## Content & Language Strategy

**支持语言：** 简体中文（主）+ English（备）

**叙事策略：**
- 计数器用"识别"而非"处理"——更有画面感，暗示 AI 在"看"每笔交易
- 完成文案"消费全貌"呼应 01.5 标题"看看过去几个月的消费全貌"，形成叙事闭环
- 错误状态不用"失败"这种词，用"遇到问题" + "部分未能识别"，降低挫败感
- 返回弹窗的"继续等待"是默认高亮按钮，引导用户留下

**语言：** 界面简体中文，代码/后端英文

## On-Page Interactions

| # | 交互 | 触发 | 行为 | 反馈 | 类型 |
|---|------|------|------|------|------|
| 1 | 页面进入 | 从 01.5 自动跳转 | 初始化动画序列：AI 大脑淡入 -> 脉冲开始 -> 计数器从 0/{total} 开始 | 页面 fade-in（300ms） | 系统 |
| 2 | AI 分类进行中 | 后端持续返回分类结果 | 三个同步动画：(a) Flying Labels 从底部飞入 AI 图形；(b) Category Bubbles 生长/新增；(c) Live Feed 推入新条目 | 计数器递增 + 进度条推进 | 系统 |
| 3 | 新分类首次出现 | 后端返回此前未出现的分类 | 新气泡从 AI 图形中"弹出"（scale 0->1, spring easing），附带分类名和图标 | 轻微震动反馈（haptic, light） | 系统 |
| 4 | 已有分类追加 | 后端返回已有分类的新交易 | 对应气泡体积增大（smooth resize），Flying Label 飞入该气泡 | 气泡脉冲一次（scale 1.05->1.0） | 系统 |
| 5 | 进度更新 | 每条交易处理完成 | 计数器 {current} 递增（数字滚动动画），进度条同步推进 | 视觉连续性 | 系统 |
| 6 | 处理完成 | current == total | 暂停 500ms -> 所有动画 fade-out（400ms）-> 完成庆祝视图 fade-in：成功图标 draw -> 结果汇总 stagger-in -> 分类标签逐个弹入 -> 过渡文案淡入 | 成功震动反馈（haptic, success） | 系统 |
| 7 | 自动跳转 | 完成庆祝视图展示 2.5s 后 | navigate to 01.7 Dashboard | 页面右滑过渡 | 系统 |
| 8 | 返回手势/按键 | 用户按返回键或左滑（非完成状态） | 弹出确认弹窗（"AI 正在处理中"） | 弹窗从底部滑入 | 用户 |
| 9 | 返回手势/按键 | 用户按返回键或左滑（完成庆祝状态） | 不弹窗，直接跳转 Dashboard | 页面淡出 | 用户 |

## Animation Timing Rules

**动画循环机制：** 动画持续循环播放，无硬性时间上限，直到后端处理真正完成。避免"动画放完了但还没处理好"的尴尬。

```
T+0.0s    页面进入，AI 大脑淡入
T+0.5s    脉冲开始，计数器出现 "正在识别 0/{total} 笔交易..."
T+1.0s    第一条 Flying Label 飞入，第一个气泡诞生
T+1.0s~   持续动画周期，循环播放直到后端完成
          ↳ 最低展示时长保证：即使后端 2s 完成，前端动画也拉伸到至少 5s
          ↳ 少量交易（<10笔）：节奏放慢到 1.5-2s/条，保证最低 5s
          ↳ 大量交易（>500笔）：抽样展示，加速节奏到 0.3-0.5s/条
T+finish  处理完成 -> 500ms 暂停 -> 庆祝视图过渡
T+finish+2.5s 自动跳转 Dashboard
```

**前端动画 vs 后端处理的解耦策略：**
- 后端可能在 2-3 秒内完成所有分类
- 前端维护一个"待展示队列"，按节奏逐条播放，循环直到后端完成
- 进度条的"真实进度"和"展示进度"解耦：展示进度匀速推进，真实进度到 100% 时展示进度加速追上
- 保证动画流畅且不会出现"卡在 99%"的尴尬

**状态机：**

```
                    ┌──────────┐
                    │  Enter   │
                    └────┬─────┘
                         │
                         ▼
              ┌──────────────────┐
              │  Initializing    │  0.5-1s
              └────────┬─────────┘
                       │
                       ▼
              ┌──────────────────┐
              │   Processing     │ ◄──── 循环动画
              │  (主状态)         │       直到后端完成
              └────┬───────┬─────┘
                   │       │
             完成  │       │ 错误/超时
                   ▼       ▼
          ┌──────────┐  ┌──────────┐
          │Completion │  │  Error   │
          │ (庆祝)    │  │ (错误)   │
          └────┬──────┘  └──┬───┬──┘
               │            │   │
         2.5s  │     重试   │   │ 跳过
         自动  │            │   │
               ▼            │   ▼
          ┌──────────┐     │  ┌──────────┐
          │Dashboard │ ◄───┘  │Dashboard │
          │ (01.7)   │       │ (部分数据) │
          └──────────┘       └──────────┘

  * 任意非完成状态下按返回键 -> Back Dialog 弹窗
  * 完成状态下按返回键 -> 直接跳转 Dashboard
```

**完整状态定义：**

| State | When | 视觉表现 | 可用操作 | 持续时间 |
|-------|------|---------|---------|---------|
| **Initializing** | 页面进入，等待后端首个响应 | AI 大脑淡入 + 脉冲，计数器 "正在识别 0/{total} 笔交易..."，进度条 0% | 返回键 | 0.5-1s |
| **Processing** | 后端持续返回分类结果 | Flying Labels + Category Bubbles + Live Feed 循环动画，计数器递增，进度条推进 | 返回键 | 5s 最低保证，无上限（循环） |
| **Processing (Few)** | total < 10 笔交易 | 同 Processing，动画节奏放慢 1.5-2s/条，保证最低 5s | 返回键 | 5-8s |
| **Processing (Slow)** | 30s 内计数器无变化但连接仍在 | 动画继续循环，出现提示"处理量较大，请稍候..." | 返回键 | 直到恢复 |
| **Processing (Stalled)** | 60s 完全无后端响应（连接断开） | 自动重连 3 次（2s/4s/8s）-> 失败则转入 Error | 返回键 | 最长 ~74s |
| **Completion** | current == total，全部处理完成 | 全屏替换为庆祝视图 | 返回键（直接跳转） | 2.5s 后自动跳转 |
| **Completion (Low Coverage)** | 覆盖率 < 80% | 同 Completion，额外提示"部分交易需要你来帮忙分类" | 返回键（直接跳转） | 2.5s 后自动跳转 |
| **Error (Partial)** | 部分交易处理失败 | 错误视图 + "部分交易未能识别" | 重试 / "先看已识别的" / 返回键 | 用户操作 |
| **Error (Total)** | 整体处理失败或连接超时 | 错误视图 + "处理遇到问题，请检查网络" | 重试 / 返回键 | 用户操作 |
| **Back Dialog** | 用户任意非完成时刻按返回键 | 底部弹窗 | 去首页 / 继续等待 | 用户操作 |

**Stalled 判定逻辑：**
- 连接仍在 + 有心跳但无新分类结果 -> **不是 Stalled**，只是"慢"，30s 后显示善意提示
- 连接断开 / 完全无响应 -> **真正的 Stalled**，尝试重连，失败则 Error
- 关键判断依据是后端连接是否还活着，而不是分类速度

**补充状态说明：**

| 项目 | 说明 |
|------|------|
| Slow 提示 ZH | "处理量较大，请稍候..." |
| Slow 提示 EN | "Processing a large batch, hang tight..." |
| Low Coverage 提示 ZH | "部分交易需要你来帮忙分类" |
| Low Coverage 提示 EN | "Some transactions need your help to categorize" |
| 后端继续处理 | 用户通过返回键或错误跳过离开后，后端不中断处理。Dashboard 展示当前已有数据，后续静默更新 |
| 再次进入 | 用户离开后不会回到此页面。此页面仅在上传后自动触发一次 |

## Data Validation

| # | 验证对象 | 规则 | 触发时机 | 失败处理 |
|---|---------|------|---------|---------|
| 1 | 入口参数 | 必须携带有效的 upload_id | 页面初始化 | 缺少 upload_id -> redirect to 01.7 Dashboard |
| 2 | 后端首次响应 | 10s 内必须收到后端第一条结果或 total 笔数 | Initializing | 10s 无响应 -> "连接中..."提示，30s 仍无 -> Error |
| 3 | total 合理性 | total > 0 | 收到 total 值时 | total == 0 -> "未找到交易记录，请确认文件内容"+ 返回 01.5 或跳转 Dashboard |
| 4 | 分类结果完整性 | 每条须含 transaction_id, merchant, amount, category | 每条结果到达时 | 缺少字段的交易静默跳过，不展示在 Live Feed，但计入进度 |
| 5 | 覆盖率计算 | coverage = recognized / total * 100 取整 | 处理完成时 | coverage < 80% -> Completion (Low Coverage) |
| 6 | 连接状态 | WebSocket / SSE 连接存活 | 持续检测 | 断开 -> 自动重连 3 次（2s/4s/8s）-> 失败 -> Error (Total) |

**边界情况：**

| 边界 | 处理 |
|------|------|
| CSV 只有 1-2 笔交易 | 最低 5s 动画，节奏放慢至 1.5-2s/条 |
| CSV 有上千笔交易 | 循环播放，Live Feed 抽样展示，进度条匀速 |
| 未知分类出现 | 显示为"其他"类别，气泡使用默认灰色 |
| 完成庆祝期间按返回键 | 不弹确认框，直接跳转 Dashboard |
| 网络恢复后重新连接 | 从断点继续，计数器跳到真实进度，动画快速追上 |

## Design Decisions

1. **零操作页面** — 大强什么都不用做，只需要看。唯一的"退出"是处理完成后自动跳转
2. **信任建立剧场** — 动效要张扬、有视觉冲击力。这是核心产品承诺的验证时刻
3. **动画循环机制** — 不设硬性时间上限，循环播放直到后端完成，避免"动画放完了但还没处理好"
4. **最低展示时长 5s** — 即使后端秒级完成，动画至少展示 5 秒以确保信任建立
5. **前后端解耦** — 前端"待展示队列"按节奏播放，展示进度与真实进度解耦
6. **返回键处理** — 非完成状态弹窗确认，完成状态直接跳转。后端始终继续处理
7. **Stalled 判定** — 基于连接是否存活，而非分类速度。连接在 = 只是慢，连接断 = 真 Stalled
8. **Category Summary 保留** — 维度是"本次导入"，不同于 Dashboard 的"所有数据"全貌
9. **Haptic 反馈** — 新分类轻震、完成成功震、错误警示震，增强物理感知

## Technical Notes

- **数据通信：** WebSocket 或 SSE（Server-Sent Events）实时接收分类结果
- **重连策略：** 指数退避重连（2s/4s/8s），最多 3 次
- **待展示队列：** 前端维护 FIFO 队列，后端结果入队，动画按节奏出队
- **动画库：** React Native Reanimated 2 实现复杂动画（气泡 + 飞入 + 进度）
- **Haptic：** Expo Haptics API (`Haptics.impactAsync`, `Haptics.notificationAsync`)
- **进度解耦：** 真实进度 state 和展示进度 animated value 分离，完成时展示进度加速追上真实进度
