## Overview

**Page Purpose:** 引导用户配置快速触发记账方式，覆盖三种触发机制：Siri 语音（iOS Shortcuts）、Back Tap / Action Button（iOS 系统辅助功能）、截屏分享识别（Share Extension，跨平台）。这是"零记账"理念的极致延伸 -- 连打开APP都不需要。

**User Situation:** 大强（或任意 iOS 用户）在分类管理或设置页看到"快速触发"入口，好奇点进来。他可能听说过 Siri 快捷指令但从没用过，或者在小红书上看到过截屏记账的玩法，需要一步步教他怎么设置适合自己场景的触发方式。

**Entry Points:**
- 06.2 Category Management 页面底部入口（仅 iOS 设备显示）
- 08.1 Settings 页面"快速触发记账"入口（仅 iOS 设备显示）

**Exit Points:**
- 配置完成 -> 返回来源页面
- 关闭/返回 -> 返回来源页面

### Cross-Platform Note

本页面仅在 iOS 设备显示。但"截屏分享识别"能力是跨平台的（iOS Share Extension / Android Share Intent），Android 端不通过本页面暴露，而是通过以下入口：
- 06.1 手动记账页顶部 tip："刚付完款？截屏分享给我，自动识别"
- 08.1 设置页"截屏记账"开关（启用后，App 出现在系统分享菜单中）

Phase 3 若 Google Assistant Routines 成熟，可为 Android 开放本页面并增加语音触发方式。

## Core Experience Design

### 设计理念

三种触发方式各有适用场景，技术机制各不相同，用户按需选择配置。每种方式都提供分步图文教程 + 测试验证。目标是让从未用过这些功能的用户也能在 2 分钟内完成配置。

**触发方式对照：**

| 触发方式 | 典型场景 | 是否出声 | 技术机制 | 配置位置 |
|---------|---------|---------|---------|---------|
| 🎤 Siri 语音 | 开车、做饭、双手占用 | 需要 | iOS Shortcuts (SiriKit + INShortcut) | App 内一键安装 → Siri 自动注册 |
| 📲 Back Tap / Action Button | 地铁、会议后、安静场合 | 不需要 | iOS Shortcuts + 系统辅助功能 | App 内安装 Shortcut → iOS 设置中绑定触发方式 |
| 📸 截屏分享识别 | 刚付款，支付页还在屏幕上 | 不需要 | Share Extension + OCR（跨平台） | App 安装后自动注册 Share Extension，无需额外配置 |

### 核心功能点

| 功能 | 说明 |
|------|------|
| **三种触发方式 Tab** | 页面顶部 Tab 切换：语音 / 按键 / 截屏。同一时间只显示一个 Tab 内容，用户按需选择配置 |
| **预设快捷指令** | 语音方式提供 2 个预设 Shortcuts 模板，一键安装；按键方式提供 1 个 Shortcut + 系统设置引导 |
| **分步图文教程** | 每种方式 3 步配图引导，配 iOS 系统截图示意。文字口语化，不用技术术语 |
| **一键安装** | 语音指令通过 App 内 INVoiceShortcutCenter 注册或 iCloud 分享链接安装 |
| **截屏分享识别** | 无需额外配置。App 安装后自动注册 Share Extension，用户截屏后通过系统分享菜单触发，复用 01.5 OCR 能力（单笔即时场景） |
| **测试验证** | 语音和按键方式安装后提供测试入口，用户可立刻试用并确认功能正常 |

### 预设快捷指令清单

| 指令名 | 默认触发方式 | 功能 | 参数 |
|--------|------------|------|------|
| 语音记账 | Siri: "记一笔{描述}{金额}" | 通过 Siri 语音创建一笔消费记录 | 描述(文本) + 金额(数字) |
| 快速记固定消费 | Siri: "记通勤打车" | 记录预设金额的固定消费 | 用户自定义金额+分类 |
| 按键弹出记账 | Back Tap 双击 / Action Button | 静默触发，直接弹出手动记账页 | 无 |
| 截屏记账 | 截屏后 Share Sheet 分享给 App | 识别截图中的支付金额和商户，进入确认页 | 无（OCR 自动提取） |

## Content Strategy

**语气：** 友好、鼓励、手把手教。像朋友教你用新功能。

**关键文案方向：**
- 语音 Hero: "嘿Siri，帮我记一笔" -- 直接展示最终效果
- 按键 Hero: "双击背面，弹出记账" -- 强调静默、快速
- 截屏 Hero: "截屏就是记账" -- 最低操作成本
- 步骤标题: 每种方式 3 步，极简（"安装 → 绑定 → 试一试"）
- 测试: 给具体示例（"试试说：嘿Siri，记一笔午饭38块"、"双击手机背面试试"）

**禁用词：** "URL Scheme""Shortcuts API""自动化"（技术术语）

---

## Key Design Decisions

1. **三种触发方式并列，技术机制各异** -- Siri 语音是 iOS Shortcuts 体系；Back Tap / Action Button 是 Shortcuts + 系统辅助功能；截屏分享是 Share Extension + OCR。三者配置流程完全不同，Tab 结构让每种方式的教程独立
2. **预设指令优先于教用户自建** -- 绝大多数用户不会自己从零配置 Shortcuts。提供预设模板 + 一键安装是唯一可行的路径
3. **本页面 iOS 显示，截屏分享跨平台另行暴露** -- 语音和按键是 iOS 专属能力，页面仅 iOS 显示。截屏分享能力跨平台，Android 端通过 06.1 手动记账页 tip 和 08.1 设置页暴露，不走本页面
4. **图文教程而非视频** -- 教程页面需要快速扫描，图文比视频更高效。每步一张 annotated screenshot + 一句话说明
5. **截屏识别复用 01.5 OCR 能力** -- 不重复建设。01.5 是批量导入场景，06.3 截屏是单笔即时场景，入口不同但底层 OCR 引擎相同
6. **截屏分享无需额外配置** -- 与语音/按键不同，Share Extension 随 App 安装自动注册，用户无需任何设置步骤。教程只需告知"怎么用"而非"怎么装"
7. **测试环节必须有** -- 用户配置完不确定是否成功，一个"开始测试"按钮让他立刻试用并获得信心

## Implementation Notes (Phase 2)

- **依赖：** iOS Shortcuts framework (INShortcut) + SiriKit donation（语音）；Back Tap / Action Button 通过 iOS Shortcuts 触发 URL Scheme（按键）；Share Extension + OCR（截屏）
- **最低版本：** iOS 14+（Back Tap 需要 iOS 14；Action Button 需要 iPhone 15 Pro+）
- **分发方式：** iCloud Shortcuts 分享链接（最简单）或 App 内 INVoiceShortcutCenter 注册
- **深度链接：** 快捷指令通过 URL Scheme 或 Universal Link 调用 APP 的记账接口
- **后台执行：** Siri 触发后可在后台完成记账（无需打开 APP UI），通过 Siri response 确认结果
- **Share Extension：** 截屏识别需要实现 iOS Share Extension，接收图片后调用 OCR 引擎（复用 01.5 能力）
