## Overview

**Page Purpose:** "3秒补一笔" -- 让用户通过语音/表单/截图OCR三种方式快速补充AI未捕获的消费记录。核心设计目标是把"手动记账"做到不像手动记账 -- 语音说一句话或拍张照片，AI完成剩余填充，用户只需确认保存。

**User Situation:** 大强在菜市场花了50块现金买菜，没有电子支付记录。回到家想起来，打开APP点"+"补一笔。他害怕又要填一堆字段（金额、分类、日期、备注），但如果只需要说"菜市场50块"就搞定，他愿意试试。

**Success Criteria:**
- 语音模式从唤起到保存 <= 5s（含AI处理时间）
- 表单模式从打开到保存 <= 10s（仅填金额+确认分类）
- 手动记账放弃率 <= 15%（打开页面后未保存就退出）
- AI分类首次准确率 >= 75%（用户无需手动修改分类）

**Entry Points:**
- Dashboard 底部 "+" FAB（01.7）
- Transaction List 底部 "+" FAB（03.1）
- iOS Shortcuts / Siri 语音触发（06.3，Phase 2）

**Exit Points:**
- 保存成功 -> 返回来源页面（Dashboard / Transaction List）
- 保存后点击"调整分类" -> 06.2 Category Management
- 关闭/取消 -> 返回来源页面（不保存）

## User Mental State

| 维度 | 状态 |
|------|------|
| 触发事件 | 想起有一笔现金消费（菜市场50块）没被APP记到，点了"+" |
| 情绪基调 | 轻微不耐烦 + 试探（"又要填表吗？"） |
| 希望 | "说句话就搞定，别让我选分类、填日期、写备注" |
| 恐惧 | "如果要填5个字段我就不记了，50块钱不值得花1分钟" |
| 内心问题 | "我只需要说金额和地方就够了吧？""AI能不能自动帮我分好类？" |
| 耐心程度 | 极低 -- 手动记账本身就是"零记账"承诺的反面，用户每多一秒都在质疑产品 |

## Content & Language Strategy

**核心叙事：补一笔 ≠ 手动记账**

"记一笔"而非"手动记账" -- 在措辞上弱化"手动"这个词，强调的是"补充"而非"输入"。AI 在整个流程中无处不在：语音解析、分类推荐、OCR识别，让用户感知到"我只是告诉APP一声，剩下的它搞定"。

| 维度 | 策略 |
|------|------|
| 页面标题 | "记一笔" -- 口语化、轻量，不用"添加交易""新建记录" |
| 语音提示 | "试试说：菜市场50块" -- 真实生活场景示例，不用"请说出消费金额和类别" |
| AI分类标签 | 推荐分类旁加"AI"小标签，让用户知道这是AI建议可以修改 |
| 成功反馈 | "已记录 ¥50.00 食品·生鲜" -- 简洁确认，附带"调整分类"入口 |

**语气：** 轻松、简洁、鼓励性。像朋友说"记一下就好"。

**禁用词：** "手动""输入""填写""必填"（强化负面感知的词）

**语言：** 界面简体中文，代码/后端英文

**边界情况：**
- **金额为 0 或未输入：** 保存按钮灰色不可点。不允许金额为 0 的手动记录
- **语音识别出多笔：** MVP 语音只解析单笔。说"菜市场50块和水果20块"时，取第一笔（菜市场50块），提示"目前只能记一笔，其余请重新录入"
- **截图无消费信息：** 显示"未识别到消费信息"，引导切换到表单模式
- **截图识别到的金额有误：** 用户可点击条目进入编辑（切换到表单模式预填该条目）
- **网络离线时保存：** 本地缓存记录，网络恢复后自动同步。保存时不阻塞用户
- **快速重复点击保存：** 按钮点击后立即切换为 loading 态，防止重复提交
- **日期选择超出范围：** DatePicker 仅允许选择过去 90 天到今天
- **AI 分类推荐超时：** 3s 内无推荐结果，分类行显示"选择分类"（手动选择），不阻塞保存
- **麦克风权限未授权：** 切换到语音模式时弹出权限请求。拒绝后显示"需要麦克风权限"+ "去设置"链接
- **相机/相册权限未授权：** 同上处理模式

## Design Decisions

1. **表单模式为默认** -- 尽管语音模式更快，但首次使用的用户对语音记账有认知门槛。表单是最符合预期的默认选择，语音是惊喜发现
2. **金额是唯一必填字段** -- 分类有AI推荐，日期默认今天，备注选填。把必填字段减到最少，降低D3-（害怕手动输入）的心理负担
3. **Modal 而非 Push 导航** -- 记一笔是"打断当前流程的临时操作"，Modal 的"做完就回来"语义更合适。用户不会在记账页停留太久
4. **自定义数字键盘** -- 系统键盘弹出时会导致布局跳动，而且包含不需要的字母键。自定义键盘内嵌页面底部，体验更流畅
5. **语音用"按住说话"而非持续监听** -- 按住说话给用户明确的开始/结束控制，避免误触发和隐私顾虑
6. **截图OCR支持多笔批量** -- 用户可能截图一张支付宝账单包含多笔消费，批量勾选比一笔一笔拍效率高很多
7. **成功Toast附带"调整分类"** -- 保存后的 Toast 是进入 06.2 的轻量入口。不强推，但对想纠正 AI 分类的用户提供便捷通道
8. **离线缓存保存** -- "零记账"的承诺不能因为网络不好而打折。手动记录本地先存，后台同步
9. **页面标题用"记一笔"** -- 口语化、轻量。相比"添加交易""新建记录""手动记账"，"记一笔"最接近用户内心的想法

## Technical Notes

- **AI 分类推荐：** 输入金额/备注后调用 AI 分类接口，timeout 3s。推荐基于：备注关键词匹配 > 历史消费模式 > 金额范围统计
- **语音识别：** 调用端侧或云端 ASR（推荐端侧优先减少延迟），识别结果传给 AI 做 NLU 解析（提取金额、商户、分类意图）
- **OCR 识别：** 调用云端 OCR API，识别支付截图中的交易信息（金额、商户名、时间）。需支持支付宝、微信支付、银行APP截图格式
- **本地缓存：** 使用 AsyncStorage 或 SQLite 本地存储未同步的交易记录，NetworkInfo 监听恢复后批量同步
- **金额存储：** 前端输入的元转为分（整数）存储，避免浮点精度问题
- **防重复提交：** 保存按钮点击后设置 loading flag，接口返回前不可再次点击
