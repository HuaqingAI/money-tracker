## Overview

**Page Purpose:** "你没注意到的事，AI替你注意了" -- 展示AI主动生成的消费洞察Feed，让大强发现自己没意识到的消费模式。与07.1 AI Chat的"用户主动问"不同，这里是**AI主动说** -- 不需要用户想问题，打开就有发现。

**Design Concept: 消费日报** -- 把AI洞察包装成一份每天更新的"消费日报"。每张洞察卡片是一个"发现"，而非一条"通知"。用户打开Feed的心态应该像翻看朋友圈 -- "看看今天AI发现了什么"。

**User Situation:** 大强在07.1跟AI聊了外卖的事，AI在回答末尾附了一个"查看更多洞察"按钮。大强点进来，看到AI主动为他生成的本月消费洞察。他不需要问问题 -- AI已经分析好了，等他来看。或者，大强收到一条推送"你本月外卖频率比上月高了30%"，点开推送直接进入这个页面。

**Success Criteria:**
- Feed打开率（从推送通知进入）>= 25%
- 单次会话平均浏览洞察数 >= 3条
- 洞察"有用"标记率 >= 40%
- 从洞察进入优化建议的转化率 >= 20%
- 洞察分享率 >= 5%

**Entry Points:**
- 07.1 AI Chat回答中的"查看更多洞察"按钮
- 推送通知点击（某条具体洞察）
- 底部Tab "AI" -> 顶部Tab切换到"洞察"（如果AI页面采用顶部Tab模式）
- Dashboard上的洞察入口卡片（条件显示）

**Exit Points:**
- 点击洞察卡片的"查看建议" -> 07.3 Optimization Suggestions
- 点击洞察卡片中的数据引用 -> 03.1 Transaction List / 03.3 Trend Chart
- 点击"追问AI" -> 07.1 AI Chat（预填追问内容）
- 底部Tab切换
- 返回手势

## User Mental State

| 维度 | 状态 |
|------|------|
| 触发事件 | 从AI Chat点击"查看更多洞察"，或从推送通知"你本月外卖频率比上月高了30%"点进来 |
| 情绪基调 | 好奇 + 期待（"AI还发现了什么我没注意到的？"）|
| 希望 | "有什么有趣的发现，最好是我自己不会注意到的消费模式" |
| 恐惧 | "别全是'你花了很多'这种废话"、"别像理财APP一样说教" |
| 内心问题 | "还有什么是我没注意到的？""这些数字靠谱吗？""该怎么做？" |
| 耐心程度 | 中 -- 像刷信息流一样浏览，有趣的多看一眼，无聊的划过。3条之内没有亮点就会走 |
| 关键信号 | 一条洞察如果包含"你可能没注意到"的信息（如"你本月瑞幸消费12笔 ¥216，超过去年同期3倍"），注意力会立刻被抓住 |

**心态转变设计目标：**

```
进入时          浏览中             看到行动指引后
好奇浏览  -->  发现惊讶  -->  行动意愿
"看看有啥"   "原来如此！"    "我要去调整一下"
```

## Content & Language Strategy

**核心叙事：消费日报**

洞察Feed不是"报警系统"，而是"每日发现"。每张卡片的写作策略是**陈述事实 + 提供视角**，不做评判。

| 维度 | 策略 |
|------|------|
| 标题 | 事实描述而非评判。"外卖频率环比上升"而非"你外卖吃太多了" |
| 正文 | 具体数字 + 对比维度。"40次 vs 36次 (+11%)"而非"你外卖次数增加了" |
| 变化指示 | 用橙色/青色而非红色/绿色（与03.3趋势图表一致，避免涨跌心理暗示） |
| 行动引导 | 每条洞察附带"查看明细/建议"和"追问AI"两个方向 |
| 分享 | 分享卡片默认隐藏金额（仅百分比和模式），保护隐私 |

**洞察类型及文案模板：**

| 类型 | 标题模板 ZH | 正文模板 ZH |
|------|------------|------------|
| 分类环比变化 | "{category}消费环比{direction}" | "你本月{category} {count}次，比上月{direction} {diff}次（{percent}%）。本月{category}总计 ¥{amount}。" |
| 消费模式发现 | "{brand}消费习惯" | "你平均每周买 {frequency}次{brand}，月均 ¥{amount}。{comparison_note}。" |
| 固定支出占比 | "固定支出占比{level}" | "你的固定支出（{items}）占月收入 {percent}%，{comparison_note}。" |
| 异常消费 | "检测到异常消费" | "本月有 {count}笔超过 ¥{threshold}的消费，合计 ¥{total}。{detail}。" |
| 存款趋势 | "月结余趋势" | "过去{months}个月结余分别为 ¥{list}，{trend_desc}。" |

| 类型 | 标题模板 EN | 正文模板 EN |
|------|------------|------------|
| 分类环比变化 | "{category} spending {direction} month-over-month" | "You had {count} {category} transactions this month, {direction} {diff} from last month ({percent}%). Total {category} this month: ¥{amount}." |
| 消费模式发现 | "{brand} spending pattern" | "You average {frequency} {brand} purchases per week, ¥{amount}/month. {comparison_note}." |
| 固定支出占比 | "Fixed expenses ratio {level}" | "Your fixed expenses ({items}) are {percent}% of monthly income. {comparison_note}." |
| 异常消费 | "Unusual spending detected" | "{count} transactions over ¥{threshold} this month, totaling ¥{total}. {detail}." |
| 存款趋势 | "Monthly savings trend" | "Savings over the past {months} months: ¥{list}. {trend_desc}." |

**禁用词（真相镜子原则）：** "超支""浪费""过度消费""你应该""不合理""警告"

**语言：** 界面简体中文，代码/后端英文

## Insight Categories & Scheduling

**洞察生成策略：**

| 维度 | 策略 |
|------|------|
| 生成频率 | 每天1-2条（不多于3条/天，避免信息过载） |
| 推送时机 | 晚间 20:00-21:00（用户最可能查看APP的时段） |
| 优先级排序 | 异常消费 > 环比显著变化 > 模式发现 > 固定支出分析 > 存款趋势 |
| 去重策略 | 同一分类的环比变化不在7天内重复推送 |
| 数据要求 | 至少2周的消费数据才开始生成洞察（避免数据不足时的误导） |
| 个性化 | 根据用户"有用/不相关"反馈调整洞察类型权重 |

## Design Decisions

1. **Feed而非通知列表** -- 洞察不是一堆待处理的通知，而是一份可浏览的"发现"。Feed的心智模型更轻松（"看看有什么"），通知列表的心智模型更有压力（"还有几条没处理"）
2. **事实陈述不评判** -- 标题和正文只陈述数据事实和变化方向，不做好坏评判。"外卖频率环比上升"而非"你外卖吃太多了"。保持真相镜子原则
3. **橙色/青色替代红色/绿色** -- 与03.3趋势图表的颜色策略一致。上升用橙色（中性提示），下降用青色（中性提示），避免红绿带来的涨跌/好坏心理暗示
4. **双行动按钮** -- 每条洞察提供两个方向：一是"去看数据"（查看明细/趋势/建议），二是"追问AI"。前者是行动型用户，后者是好奇型用户
5. **反馈训练AI** -- 有用/不相关的反馈不只是"用户满意度"指标，而是实际影响后续洞察生成的权重。标记"不相关"的洞察类型权重降低
6. **免费用户看1条** -- 让免费用户体验到洞察的价值（1条完整展示），同时用模糊遮罩制造好奇心（"还有7条等你看"）。比完全锁住更能驱动付费转化
7. **分享隐私保护** -- 分享卡片默认不含金额，只展示百分比和模式描述。与03.4消费雷达的分享策略一致
8. **每天1-2条节制推送** -- 洞察不是越多越好。每天超过3条会造成信息疲劳，用户开始忽略所有洞察。控制数量让每条都有分量
9. **"不相关"不删除** -- 标记不相关的卡片只是视觉降权，不从列表中移除。避免误触导致内容丢失，也让用户可以回头查看
10. **空状态引导去AI Chat** -- 洞察需要数据积累，但AI Chat可以立即使用。用"去跟AI聊聊"把没有洞察的用户引导到07.1

## Technical Notes

- **洞察生成：** 后端定时任务（每日20:00-21:00），基于用户消费数据运行分析模型，生成结构化洞察
- **洞察数据结构：** `{id, type, title, body, data_refs[], comparison?, actions[], created_at, read, feedback}`
- **推送通知：** 新洞察生成后通过FCM/APNs推送，点击携带insight_id参数
- **已读标记：** 卡片进入视口1s后标记已读，批量上报后端（debounce 3s）
- **反馈API：** `PATCH /insights/{id}/feedback` body: `{helpful: true|false|null}`
- **分享卡片生成：** 前端使用react-native-view-shot截图洞察卡片（金额区域用占位符替换），生成分享图片
- **免费用户计数：** 后端记录洞察总数，前端展示"还有 N 条"
- **分页加载：** 每页10条，按created_at降序
