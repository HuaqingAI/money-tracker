## Overview

**Page Purpose:** 展示AI基于用户真实消费数据生成的可执行省钱建议，每条附具体金额和可量化影响。这是从"发现问题"（洞察）到"采取行动"（优化）的关键转化 -- AI管家不只是告诉你花了多少，而是告诉你"怎么省"。

**User Situation:** 大强在洞察Feed里看到"本月外卖频率比上月高了30%"，心想"那到底怎么省呢？"。他点击"查看建议"进入本页。此刻他不只是好奇了，而是有了行动意愿 -- 他想要具体的、可执行的方案，而不是"少花钱"这种空洞建议。

**Success Criteria:**
- 建议查看完成率（进入页面后至少展开一条建议详情）>= 65%
- "采纳"操作率（至少标记一条为"采纳"）>= 30%
- 点击"模拟效果"转化率 >= 40%
- 已采纳建议的执行追踪回访率（7天内再次进入本页）>= 20%

**Entry Points:**
- 从 07.2 洞察Feed点击"查看建议"进入
- 从 07.1 AI对话中的回答链接进入（AI回答含"查看完整建议"链接）
- 从 Dashboard "AI管家"区域的建议提醒徽章进入

**Exit Points:**
- 点击某条建议的"模拟效果"CTA -> 进入 07.4 Simulation Comparison
- 返回（左上角 / 安卓返回键）-> 上一页
- 点击底部 Tab -> 对应页面

## User Mental State

| 维度 | 状态 |
|------|------|
| 触发事件 | 看了洞察Feed后想知道"具体怎么省" |
| 情绪基调 | 行动意愿萌芽 + 谨慎乐观（"真能省这么多？"） |
| 希望 | "给我具体的、基于我自己数据的省钱方案，而不是泛泛的'少花钱'" |
| 恐惧 | "建议太笼统没用 / 建议太极端做不到 / 感觉被APP教育怎么花钱" |
| 内心问题 | "如果少吃外卖真能省那么多？""省下来的钱一年有多少？""其他人是怎么花的？" |
| 耐心程度 | 中高 -- 有明确行动意图，愿意仔细看，但如果建议质量差会快速离开 |
| 关键信号 | 大强是被数字说服的人 -- 抽象的建议他不买账，但"每月省680元，一年8160元"这种具体数字能打动他 |

## Content & Language Strategy

**叙事框架：「从知道到做到」**

07.1 大强问了AI一个问题（好奇），07.2 他看到了消费洞察（发现），07.3 他拿到了具体方案（行动）。这一页的关键不是"告诉他花多了"（洞察已经做了），而是"告诉他怎么花少一点还不影响生活"。

| 区域 | 语气 | 核心信息 |
|------|------|---------|
| 省钱潜力卡片 | 正向激励，数字冲击 | 你有很大的优化空间（正面框架，不是"你浪费了很多"） |
| 建议标题 | 具体可行，一看就懂 | "减少外卖频率"比"优化餐饮支出"更容易理解 |
| 建议详情 | 数据支撑，不说教 | 引用真实数据（你的均价、你的频率），提供对比而非命令 |
| AI解释 | 温和建议，提供替代 | "不影响工作日午餐""周末可以试试" -- 尊重用户的生活方式 |
| 执行追踪 | 正向反馈，鼓励坚持 | "已省¥340" -- 用已省金额而非剩余差距 |
| 模拟CTA | 好奇心 + 期待感 | "看看一年能攒多少" -- 延伸到年度视野，制造期待 |

**关键策略：**
- **建议用正向框架** -- "每月可省¥680"而非"你每月多花了¥680"。优化而非批评
- **AI解释引用真实数据** -- 大强信数字不信概念，每条建议必须有他自己的真实消费数据支撑
- **建议难度渐进** -- 第一条最容易执行，最后一条可能需要较大调整。降低行动门槛
- **"暂不考虑"不是"拒绝"** -- 措辞温和，保留下月重新推荐的可能
- **执行追踪用正向反馈** -- 显示"已省多少"而非"还差多少"，保持动力

**语言：** 界面简体中文，代码/后端英文

## Design Decisions

1. **省钱潜力放顶部而非底部** -- 大强需要先看到"值得看下去"的理由。¥1,860/月的数字冲击让他有动力展开每条建议
2. **建议卡片默认折叠** -- 避免信息过载。大强可以先扫一眼所有建议的分类和金额，选感兴趣的展开
3. **每条建议独立可模拟** -- 不强迫用户采纳所有建议后才能看效果。降低行动门槛
4. **AI解释不超过3句** -- 大强是忙人，长篇大论他不看。3句话：真实数据 + 可行性 + 替代方案
5. **执行追踪用进度条而非数字** -- 视觉化的进度比"减少2次/目标4次"更有激励感
6. **"暂不考虑"而非"忽略/拒绝"** -- 语气温和，不给用户拒绝建议的心理负担。AI管家是顾问不是老师
7. **省钱金额用绿色** -- 绿色在财务场景天然关联"正向" -- 这里不适用03.x的橙/蓝绿策略，因为语境不同：03.x是"消费真相"需要中性，07.3是"省钱激励"需要正向
8. **建议按可省金额降序** -- 最大的优化空间排最前，抓住注意力。如果最大建议就能省500+/月，用户更可能行动
9. **最少3个月数据才给个性化建议** -- 1-2个月数据样本太小，建议可能不准确。不足3月时给通用建议 + 积累提示

## Technical Notes

- **建议生成：** 后端 AI 分析近 3-6 个月消费数据，识别高频可优化项。每月1号自动重新生成，用户也可手动刷新
- **省钱金额计算：** (当前月均频率 - 建议频率) x 均价 = 月省金额。年省 = 月省 x 12
- **执行追踪：** 基于本月实际消费数据与采纳目标对比，自动计算进度。不需要用户手动打卡
- **建议去重：** 同一分类不重复给建议（不会同时给"少吃外卖"和"降低外卖均价"）
- **数据缓存：** 建议数据按月缓存本地，执行追踪数据实时更新
- **骨架屏：** 潜力卡片用数字占位 + 脉冲动画，建议卡片用标准卡片骨架
- **采纳状态持久化：** 存储在用户 profile 中，跨设备同步
