## Overview

**Page Purpose:** "你的财务数据会说话" -- 让大强用自然语言向AI管家提问，获得基于**自己真实消费数据**的量化回答。这不是通用聊天机器人，而是一个读过你所有账单的私人财务顾问。核心差异：每一个回答都锚定在用户的真实数据上，而非泛泛的理财建议。

**Design Concept: 数据对话** -- AI管家的回答必须包含具体数字、具体分类、具体时间段。"你3月外卖花了1,280元，平均每次32元，如果每月减少4次可省128元/月，年省1,536元"。泛泛回答（"少点外卖能省钱"）= 产品失败。

**User Situation:** 大强用了3个月Money Tracker，周日下午在家，闲着无聊打开APP。他最近感觉钱花得不少但说不清花在哪，想问问AI"如果每月少吃一次外卖，一年能省多少"。他不是在寻求理财课程，而是在跟一个了解他消费习惯的"人"聊天。

**Success Criteria:**
- 首次回复延迟 <= 3s（流式输出，首token <= 1s）
- 回答中包含用户真实数据引用率 >= 90%（非泛泛建议）
- 单次会话平均问答轮数 >= 2.5（用户持续追问 = 信任建立）
- 免费体验用户转付费率 >= 8%
- 对话后进入洞察Feed的转化率 >= 15%

**Entry Points:**
- 底部Tab "AI"（日常入口）
- Dashboard AI覆盖率区域的"问问AI管家"入口（条件显示，付费用户）
- 推送通知（洞察推送中的"追问"按钮点击）
- 07.2 Insights Feed 中某条洞察的"追问AI"按钮
- 深度链接

**Exit Points:**
- 点击AI回答中的"查看更多洞察" -> 07.2 Insights Feed
- 点击AI回答中的"查看优化建议" -> 07.3 Optimization Suggestions
- 点击AI回答中的数据引用 -> 03.1 Transaction List（带筛选参数）
- 底部Tab切换到其他页面
- 返回手势/返回键

## User Mental State

| 维度 | 状态 |
|------|------|
| 触发事件 | 周日下午闲着，想到"最近外卖是不是花太多了"，点开AI Tab |
| 情绪基调 | 好奇 + 轻松（"聊聊看，反正闲着"）|
| 希望 | "给我一个具体数字，别跟我说'少吃外卖能省钱'这种废话" |
| 恐惧 | "别是个智障机器人，问什么都答不上来"、"别给我推销付费功能" |
| 内心问题 | "少吃外卖到底能省多少？""我跟同龄人比花得算多吗？""存款缺口到底多大？" |
| 耐心程度 | 中高 -- 周日有时间，愿意多聊几轮，但首次回答如果太水就不会继续 |
| 关键信号 | AI第一句话如果包含"你3月外卖花了1,280元"这样的具体数字，信任立刻建立 |

**心态转变设计目标：**

```
进入时          首次回答后         多轮对话后
好奇试探  -->  惊喜信任  -->  依赖习惯
"能聊啥？"   "哇它真的知道！"  "我想问问..."
```

## Content & Language Strategy

**核心叙事：你的数据会说话**

AI管家不是一个通用聊天机器人。它的每一句话都必须锚定在用户的真实数据上。这是"从工具到顾问"跃迁的关键 -- 顾问之所以有价值，是因为他了解你的具体情况。

| 维度 | 策略 |
|------|------|
| AI开场 | 第一句话包含用户名和数据概览（"你好大强，你3月消费共¥12,847"），立即建立"它知道我"的感觉 |
| 数据引用 | 每个回答必须包含至少一个具体数字，引用来源可追溯（可点击跳转交易列表） |
| 追问引导 | 每个回答后附带2-3个追问建议Chip，降低"不知道再问什么"的断裂感 |
| 行动导向 | 回答末尾附带行动按钮（查看明细/查看建议/模拟存款），AI不只是回答问题，还引导下一步 |
| 语气 | 专业但温暖，像一个了解你的朋友而非银行顾问。不说"您"，说"你" |
| 禁用 | 不使用"超支""浪费""过度"等评判性语言（真相镜子原则）；不给出超出数据支撑的推断 |

**AI回答结构模板：**

```
[数据陈述] 根据你过去N个月的数据...
[数据展示] 内嵌数据卡片（具体数字、图表）
[分析结论] 如果...那么...（量化的、可执行的）
[追问引导] "要不要看看...""如果...呢"
[行动按钮] [查看明细] [更多建议]
```

**免费体验文案策略：**
- 不在第一次对话时提及付费（让用户先感受价值）
- 第二次对话后轻提"你还有1次免费体验"
- 用完后用"礼物化"语言引导（"免费领取7天体验"而非"立即购买"）

**语言：** 界面简体中文，代码/后端英文

## Design Decisions

1. **流式输出而非等待** -- AI回答采用流式输出（逐字展示），首token目标 <= 1s。用户看到文字逐渐出现的过程本身就是"AI在思考"的信任建立，比等3秒后一次性弹出整个回答体验好得多
2. **数据卡片嵌入而非纯文本** -- AI引用数据时不只是文字列举，而是用结构化的Data Card展示。视觉区分"AI的话"和"数据事实"，强化数据的权威感和可信度
3. **数据引用可点击** -- 每个具体数字都是可验证的。点击"3月外卖 ¥1,280"跳转到交易列表的餐饮筛选视图。这是AI管家区别于通用聊天机器人的核心设计 -- 不信？你自己去看
4. **免费3次体验** -- 让用户先感受到价值再谈付费。3次足够完成一个完整的"提问-惊喜-追问"循环。第一次不提付费，第二次轻提剩余次数，第三次后出现Paywall
5. **推荐问题动态生成** -- 不是固定的4个问题，而是根据用户数据特征动态排序。如果用户最近外卖消费环比涨30%，"外卖花了多少"排第一
6. **追问Chip降低断裂感** -- 每个AI回答后附带2-3个追问方向。用户不需要"想好再问"，点一下就能继续对话。这是单次会话平均2.5轮的关键设计
7. **Paywall礼物化** -- 与02.1订阅页策略一致，用"免费领取7天体验"而非"购买会员"。降低付费心理门槛
8. **AI Typing显示分析动作** -- 不是通用的"思考中..."，而是"正在分析你的外卖消费数据"。让用户感知AI确实在做与问题相关的工作
9. **对话历史持久化** -- 对话不会因为切换Tab或退出APP而丢失。用户下次进入看到上次的对话，延续"长期顾问"的关系感
10. **非财务问题温和拒绝** -- 不硬性报错，而是温和引导回财务话题，保持对话流畅感

## Technical Notes

- **AI接口：** 流式响应（Server-Sent Events），携带用户ID和对话上下文
- **对话历史：** 本地SQLite存储 + 云端备份。本地优先加载，后台同步
- **数据卡片渲染：** AI返回结构化JSON（type: "data_card", data: {...}），前端解析为Data Card组件
- **行动按钮渲染：** AI返回结构化JSON（type: "action_buttons", items: [{label, target, params}]），前端解析为按钮组
- **追问建议：** AI返回结构化JSON（type: "followup_suggestions", items: [...]）
- **免费次数计数：** 后端记录，每次"用户发送消息 + AI完成回答"计为1次。中途失败不计
- **流式输出：** React Native中使用EventSource polyfill处理SSE
- **键盘适配：** 使用KeyboardAvoidingView + 自定义InputAccessoryView
- **深度链接参数：** `moneytracker://ai-chat?question={encoded_question}`
