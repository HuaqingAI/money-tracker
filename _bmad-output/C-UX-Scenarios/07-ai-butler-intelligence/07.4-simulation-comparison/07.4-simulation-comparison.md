## Overview

**Page Purpose:** 场景的高潮和成功时刻。两个功能合一：(1) 消费模拟（F12）-- 大强可以调整参数看"如果每月少X次外卖，一年能省多少"的动态曲线；(2) 同类对比（F13）-- 匿名对比同龄人消费水平，看到自己在人群中的位置。这一页要让大强从"可能有用"变成"我真的要这么做"。

**User Situation:** 大强刚在07.3看到"每月减少4次外卖可省680元"，他心动了但还需要一个"最后的推力"。他点击"模拟效果"想亲眼看看一年下来的存款增长曲线。而"同类对比"则给他另一种视角 -- "原来跟我差不多收入的人，餐饮只花这么多"。

**Success Criteria:**
- 模拟参数调整率（至少拖动一次滑块）>= 70%
- 模拟页完整查看率（滚动到底部）>= 60%
- Tab切换率（查看同类对比）>= 45%
- 从模拟页返回采纳建议的转化率 >= 25%
- 分享模拟结果率 >= 15%

**Entry Points:**
- 从 07.3 单条建议的"模拟效果 →"进入（携带该建议参数）
- 从 07.3 底部"模拟省钱效果"CTA进入（携带所有活跃建议参数）

**Exit Points:**
- 查看完毕，场景完成（大强获得行动动力）
- 返回（左上角 / 安卓返回键）-> 07.3
- 点击"分享我的省钱计划"-> 生成分享卡片
- 点击底部 Tab -> 对应页面

## User Mental State

| 维度 | 状态 |
|------|------|
| 触发事件 | 看到具体省钱建议后，想亲眼验证年度效果 |
| 情绪基调 | 期待 + 初步信服（"让我看看真的能省多少"） |
| 希望 | "看到一条上升的存款曲线，确认省钱是值得的" / "知道自己花得算不算多" |
| 恐惧 | "模拟出来的数字可能不够震撼" / "同类对比发现自己花得比别人多太多" |
| 内心问题 | "一年真能多存8000多？""我的消费水平在同龄人里算什么水平？""如果再多省一点呢？" |
| 耐心程度 | 高 -- 这是场景高潮，大强主动寻找"行动理由"，会认真看 |
| 关键信号 | 存款增长曲线的"上扬弧度"是最强的情绪推力。同类对比中"你比60%的同龄人更节省"是正向激励 |

## Content & Language Strategy

**叙事框架：「看见未来的自己」**

07.1 大强问了问题（好奇），07.2 看到了洞察（发现），07.3 拿到了方案（行动），07.4 他亲眼看到了"如果我这么做，未来会怎样"。消费模拟是对未来的可视化预言，同类对比是对现状的坐标定位。两者合力给大强最后的行动推力。

| 区域 | 语气 | 核心信息 |
|------|------|---------|
| 参数滑块 | 赋权感，"你来控制" | 不是AI强制你做什么，而是你自己选择省多少 |
| 存款曲线 | 期待感，正向激励 | 一条上扬的曲线比任何文字都有说服力 |
| 里程碑节点 | 具体可感，近期可达 | "3个月后就多了5580" -- 不是遥远的目标 |
| 同类对比 | 客观中性，事实呈现 | "高于68%的人" -- 不说好不好，让你自己判断 |
| 数据说明 | 透明可信，打消疑虑 | 匿名 + 脱敏 + 最低样本量，建立信任 |

**关键策略：**
- **滑块交互制造"操纵感"** -- 大强亲手拖动滑块，自己发现"原来少4次就能省这么多"。自主发现 > 被动告知
- **双线对比制造"差距感"** -- 当前趋势 vs 优化后，两条线的差距越来越大，视觉上非常有冲击力
- **里程碑拆解制造"可达感"** -- 12个月太远，3个月很近。先展示3个月效果，让大强觉得"这个我能做到"
- **同类对比用中性色调** -- 延续03.x的真相镜子原则。"你高于68%的人"用橙色（注意）而非红色（危险）
- **分享卡片不含明细** -- 只含目标总额，保护隐私。大强可以发给小芳但不会暴露具体消费

**语言：** 界面简体中文，代码/后端英文

## Design Decisions

1. **双视图Tab而非单页混排** -- 消费模拟和同类对比是两种不同的思维模式（未来预测 vs 现状定位），混在一起容易注意力分散。Tab让用户各取所需
2. **滑块而非输入框** -- 滑块制造"探索感"（"如果少一次？少两次？"），输入框是精确操作。大强此刻需要的是感性的"哇"，不是精确的计算
3. **双线面积图而非单线** -- "当前"和"优化后"的差距可视化是核心说服力。如果只有优化后的线，用户无法感受到差异
4. **里程碑用3/6/12月** -- 3个月是"很快就能感受到"，6个月是"半年效果可观"，12个月是"年度冲击"。渐进式的期待感
5. **同类对比用匿名中位数** -- 不展示排名（"你排第X名"太竞争化），用百分位（"超过62%的人"更温和）。真相镜子原则
6. **数据说明放底部而非弹窗** -- 对数据来源有疑虑的用户自然会往下滚，放底部不打扰不关心的用户
7. **分享卡片不含明细** -- 大强可能想发给小芳展示"我准备省钱了"，但不想暴露每个分类的具体消费。只展示目标总额是隐私和分享的平衡点
8. **"超过X%"措辞选择** -- "超过62%的同龄人"（你花得比62%的人多）而非"你排在前38%"（听起来像是排名竞赛）。前者是事实陈述，后者是竞争框架
9. **震动反馈配合滑块** -- 每步进一格的轻微震动让大强感受到"每一格都是真金白银" -- 触觉强化认知

## Technical Notes

- **曲线计算：** 当前趋势线 = 月均消费 x 月份数 的累计。优化线 = (月均消费 - 总省钱) x 月份数 的累计。差额 = 两线差值
- **滑块联动：** 使用 `react-native-reanimated` 的 shared values 实现滑块到曲线的实时联动，避免JS线程瓶颈
- **面积图：** 推荐 `victory-native` 或 `react-native-gifted-charts`，需支持双区域填充和渐变
- **同类对比数据：** 后端聚合匿名数据，按城市 + 收入区间 + 年龄段分桶。最小样本量 100 人
- **百分位计算：** 后端预计算各分类各分桶的百分位分布，前端直接查询
- **分享卡片生成：** 使用 `react-native-view-shot` 截取特定区域，叠加水印后导出为图片
- **数据缓存：** 同类对比数据每周更新一次（非实时），本地缓存。模拟数据基于用户最新消费实时计算
- **骨架屏：** 曲线区域显示平滑波浪线骨架 + 脉冲动画
