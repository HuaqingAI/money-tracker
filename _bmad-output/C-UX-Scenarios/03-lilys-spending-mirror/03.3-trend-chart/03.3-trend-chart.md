## Overview

**Page Purpose:** 以折线图展示近 3-6 个月消费趋势，让小丽亲眼看到"餐饮在逐月上升"的模式。这是从单笔交易到宏观模式的认知跃迁 -- 不只是"这个月花了多少"，而是"我的消费在往哪个方向走"。

**User Situation:** 小丽刚在交易详情页看到一笔瑞幸的 AI 识别详情，心里已经在想"我到底喝了多少咖啡"。她点击"查看消费趋势"进入本页，期待看到更大的画面。此刻她的好奇心大于焦虑 -- 她想知道真相，但不想被审判。

**Success Criteria:**
- 趋势图查看完成率（进入页面后至少切换一个分类筛选）>= 60%
- 分类筛选使用率 >= 50%
- 页面停留时间 >= 20s（用户在认真看趋势）
- 点击"查看消费雷达"转化率 >= 40%

**Entry Points:**
- 从 03.2 交易详情页点击"查看消费趋势"进入
- 从 01.7 Dashboard 点击趋势相关入口进入
- 从 01.8 月度报表中分类卡片的环比数字点击进入

**Exit Points:**
- 点击底部"查看消费雷达"CTA -> 进入 03.4 Spending Radar
- 点击某月数据点 -> 进入该月交易列表（03.1，筛选为对应月份）
- 返回（左上角 / 安卓返回键）-> 上一页

## Content & Language Strategy

**叙事框架：「从点到线」**

03.1 看到单月流水（点），03.2 看到单笔详情（点的微距），03.3 把这些点连成线 -- 小丽第一次看到自己消费的"方向"。折线图不只是数字，是一个趋势故事：餐饮在爬坡，购物在回落，交通稳如磐石。

| 区域 | 语气 | 核心信息 |
|------|------|---------|
| 总消费折线图 | 客观呈现，数据说话 | 你的消费总量在走什么方向 |
| 分类趋势 | 聚焦发现，中性描述 | 餐饮在逐月上升 -- 不说好不好，让你自己判断 |
| 环比变化 | 温和陈述，提供原因 | "较上月多了380元，主因外卖增加" -- 事实+原因，不给结论 |
| 月度对比 | 直观对比，一目了然 | 哪个月花得多哪个月花得少 |
| CTA 引导 | 轻度好奇心驱动 | "看看你的消费雷达" -- 激发下一步探索欲 |

**关键策略：**
- **趋势描述用中性词** -- "逐月上升""有所回落""保持平稳"，不用"暴涨""大幅下降"等情绪词
- **颜色不做道德判断** -- 上升用橙色（注意）不用红色（危险），下降用蓝绿（不错）不用绿色（奖励）
- **环比原因一句话** -- AI 生成的简短原因，帮助小丽理解"为什么变了"，但不给建议
- **CTA 用好奇心驱动** -- "看看你的消费雷达"比"分析你的消费"更有吸引力

**语言：** 界面简体中文，代码/后端英文

**边界情况：**
- **仅一个月数据：** 折线退化为单点，文案引导"下个月就有趋势了"，隐藏环比卡片，但月度对比和分类分布仍可用
- **某分类仅出现部分月份：** 折线中断月份不画点，用虚线连接相邻有数据的点
- **分类数量很少（2-3个）：** Chips 不需要横向滚动，正常排列
- **分类数量很多（>8个）：** Chips 可横向滚动，右侧渐变指示有更多
- **从 01.8 带入分类参数：** 进入时自动选中对应分类 Chip，展示该分类趋势
- **数据加载中用户返回：** 取消请求，正常返回
- **金额数值差异极大（如房贷+奶茶）：** Y 轴自适应缩放，小金额分类切换时 Y 轴重新计算
- **当月数据不完整（月中）：** 当月数据点标注"截至{day}日"，虚线连接到月末（预测区域淡色）

## Design Decisions

1. **折线图而非面积图** -- 折线更清晰展示趋势方向，面积图在多分类叠加时容易混乱。单分类时加浅色渐变填充增强视觉感受
2. **分类筛选用 Chips 而非下拉** -- Chips 一目了然、可见所有选项。小丽是"3秒决策"型用户，下拉需要两步操作（打开+选择），Chips 一步到位
3. **环比变化卡片附带 AI 原因** -- 不只告诉小丽"多了380元"，还告诉她"为什么"。一句话原因比光秃秃的数字更有信息价值
4. **颜色策略：橙+蓝绿而非红+绿** -- 避免红绿灯式的道德审判。消费上升不等于"坏"（可能是搬家月），下降不等于"好"（可能只是还没花）。真相镜子原则
5. **月度对比用水平条形图** -- 垂直条形图在手机上宽度受限，水平条形图金额标签可以自然放在条形右侧，阅读更顺畅
6. **CTA 引导到消费雷达** -- 趋势图是"分析"，雷达图是"画像" -- 从理性到感性的自然过渡。CTA 用好奇心措辞而非指令式
7. **时间范围默认近3月** -- 小丽导入数据可能只有几个月。近3月是最安全的默认值，大多数用户都能看到有意义的趋势
8. **Top 3 叠加线（全部模式）** -- 选"全部"时不显示所有分类（太乱），只显示金额最高的3个分类趋势线，让小丽快速发现"哪个在涨"
9. **单月数据不显示"数据不足"警告** -- 参照 01.8 的策略，单点也有价值（当月分类分布）。用正向引导"下个月趋势就出来了"而非"数据不足"

## Technical Notes

- **折线图组件：** 使用 `react-native-svg` + `react-native-reanimated` 实现。推荐 `victory-native` 或 `react-native-gifted-charts` 作为图表库
- **数据点触摸：** 使用 `PanResponder` 或 `Gesture Handler` 实现拖动查看数据点，计算最近点吸附
- **分类颜色映射：** 复用 01.8 定义的固定分类-颜色映射表，确保全 APP 一致
- **趋势计算：** 基于选中时间范围首末月金额对比，百分比 = (末月 - 首月) / 首月 * 100
- **环比原因生成：** 后端 AI 分析同分类内子项变化，生成一句话摘要（同 01.8 AI 洞察机制）
- **数据缓存：** 趋势数据按月缓存本地（同 01.8 策略），切换时间范围时优先读缓存
- **骨架屏：** 折线图区域显示波浪线骨架 + 脉冲动画，分类 Chips 显示圆角矩形骨架
- **金额单位简写：** >= 10000 显示为 "X.Xw"（万），>= 1000 显示为 "X.Xk"（千）。图表内使用简写，卡片内使用完整金额
