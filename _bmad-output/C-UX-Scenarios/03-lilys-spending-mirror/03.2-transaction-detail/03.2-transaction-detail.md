## Overview

**Page Purpose:** "单笔消费显微镜" -- 展示一笔交易的 AI 识别详情（品牌、分类、子分类、时间），让小丽看到 AI 的"理解能力"。同时提供分类修正和备注功能，作为 AI 识别不准时的人工兜底。这是 L1+（消费真相）在单笔粒度的呈现。

**User Situation:** 小丽在交易列表里看到一笔"瑞幸咖啡 -¥18"，点进来想确认这笔消费的细节。她的心态有两种：
- **确认型：** "AI 认出来是瑞幸了？不错，分类也对。"看一眼就返回。
- **探索型：** "让我看看这笔到底是什么... 哦原来是那天下午的冰美式。"顺便看看能不能加个标签。

**Success Criteria:**
- 详情页平均停留时间 5-15s（看完核心信息即可，不需要太长）
- 分类修改率 <= 5%（说明 AI 分类准确率高）
- "查看该品牌消费趋势" 点击率 >= 10%（引导到 03.3 的转化）

**Entry Points:**
- 从 03.1 Transaction List 点击某笔交易进入
- 从搜索结果点击某笔交易进入
- 推送通知点击（如"检测到一笔新消费"）

**Exit Points:**
- 返回 -> 03.1 Transaction List（保持列表状态）
- 点击"查看{brand}消费趋势" -> 03.3 Trend Chart（品牌维度）
- 点击"查看{category}消费趋势" -> 03.3 Trend Chart（分类维度）

## User Mental State

| 维度 | 状态 |
|------|------|
| 触发事件 | 在交易列表看到一笔有兴趣的消费（如"瑞幸咖啡 -¥18"），点击查看 |
| 情绪基调 | 好奇 + 平静（"让我看看细节"） |
| 希望 | "快速确认这笔消费的信息是对的，最好还能看到点有趣的东西" |

## Content & Language Strategy

**核心叙事：消费显微镜**

详情页的文案策略是"结构化事实" -- 用清晰的字段呈现一笔交易的所有维度。不添加任何评价性语言。AI 识别的品牌名和分类就是最好的"证据"，让小丽自己去感受。

| 维度 | 策略 |
|------|------|
| 商户名 | AI 识别的品牌名放在最显眼位置 -- 这是 AI 能力的直接展示 |
| 分类 | "餐饮·饮品" 两级分类，既准确又不过度细分 |
| 洞察卡片 | "本月瑞幸 12 笔 ¥216" -- 从个体到群体的视角切换，L2+ 的微观触发 |
| 原始摘要 | 保留原始文本让用户自行对照，增强 AI 可信度 |
| 备注 | "点击添加" 用浅灰色，不催促。备注是可选项，不是必填项 |

**语气：** 中性、结构化。像一张收据，不像一封警告信。

**禁用词：** "超支""浪费""不必要的消费"

**语言：** 界面简体中文，代码/后端英文

- **修改分类后洞察卡片更新：** 分类修改后，分类洞察卡片实时更新为新分类的汇总数据。如果品牌洞察存在，保持不变（品牌不受分类影响）

- **金额为 0 的交易：** 正常展示，Hero 金额显示"¥0"
- **超长商户名：** 最多 2 行，超长省略号截断
- **超长原始摘要：** 最多 3 行，超长省略号截断
- **该品牌本月仅 1 笔消费：** 品牌洞察卡片不显示（无汇总价值）。"本月仅此 1 笔"不需要展示
- **备注保存失败：** Toast 提示"保存失败，请重试"，编辑器保持打开
- **网络断开时修改分类/备注：** 乐观更新（先更新 UI），后台队列同步。下次联网时自动同步

## Design Decisions

1. **详情页无底部 Tab Bar** -- 详情页是"深入查看"的层级，与 Tab 页面是不同的导航深度。隐藏 Tab Bar 让内容区域更大，也暗示用户"你在往深处看"
2. **Hero 区域垂直居中** -- 商户名 + 金额 + 时间是"一眼三要素"。垂直居中的大字号呈现确保 3 秒内完成确认。这是小丽耐心度低的直接回应
3. **原始摘要保留展示** -- 让用户能对照 AI 识别结果和原始数据。"瑞幸咖啡"是 AI 从"瑞幸咖啡门店-中关村店"中提取的品牌名，展示原始文本增强 AI 可信度
4. **分类修改用 Bottom Sheet** -- 全屏切换太重（小丽只是想改个分类），内联下拉太窄（分类有两级）。Bottom Sheet 是移动端"轻量编辑"的最佳实践
5. **选择子分类自动确认** -- 减少点击次数。选了"餐饮 > 饮品"就自动关闭选择器，不需要再点"确认"。L1-（不想花精力 15/15）决定了每一步都要尽量减少操作
6. **洞察卡片从个体到群体** -- "这笔 ¥18" → "本月瑞幸 12 笔 ¥216"。从单点到模式的视角切换是 L2+（发现隐藏消费模式 13/15）的微观触发器。品牌汇总比分类汇总更有冲击力（"我竟然买了 12 杯瑞幸"比"餐饮 28 笔"更有画面感）
7. **删除按钮放最底部 + 二次确认** -- 删除是低频操作且不可逆。放在最底部避免误触，二次确认是必须的安全网
8. **备注不催促** -- "点击添加"用浅灰色，不用"请输入备注"等命令式语气。备注是增值功能，不是必填项。零输入原则意味着能不让用户写字就不让用户写字
9. **支出金额中性色** -- 与 03.1 一致，详情页的金额也不用红色。整个 Scenario 03 贯彻"真相镜子而非法官"原则
10. **数据来源透明化** -- 展示"账单导入"/"通知读取"/"手动录入"让用户理解数据的来龙去脉，增强对 AI 系统的信任

## Technical Notes

- **数据需求：** 单笔交易详情 + 该品牌本月汇总（笔数、总额）+ 该子分类本月汇总（笔数、总额），一次加载获取
- **分类修改：** 修改后应同步更新列表页和 Dashboard 的汇总数据（缓存失效）。建议后端记录用户对该商户的分类偏好，用于优化未来 AI 分类
- **备注更新：** 保存后即时生效
- **删除行为：** 用户视角不可恢复（对话框已说明）。后端实现策略（软删除/硬删除）由架构层决定
- **编辑响应性：** 分类修改和备注保存应给用户即时反馈（先更新 UI），如果后端失败需回滚并提示
- **分类数据：** 需要完整分类树（一级 + 二级），分类不频繁变化，适合缓存
- **骨架屏：** Hero 区域灰色圆形 + 文字块，Info 区域 5 行灰条
