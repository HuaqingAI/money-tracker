## Overview

**Page Purpose:** 展示个性化消费雷达图和 AI 生成的趣味洞察文案，然后生成精美可分享卡片让用户发给好友 -- 这是整个场景的高潮时刻和成功出口。雷达图不只是数据可视化，它是一张"消费画像"：六个维度勾勒出"你是什么样的消费者"。当用户截图发到群里说"我这个月奶茶竟然喝了800"时，社交裂变就启动了。

**User Situation:** 小丽刚在趋势图看到餐饮逐月上升，点进消费雷达想看"更完整的自己"。此刻她的情绪已从好奇转向期待 -- 在小红书上看到过别人晒的雷达图，现在轮到自己了。她在心里已经想好了："发到群里肯定能炸出一堆回复。"

**Success Criteria:**
- 雷达图查看完成率（进入页面后停留 >= 10s）>= 85%
- 趣味洞察阅读率（滚动到洞察区域）>= 70%
- 分享按钮点击率 >= 50%
- 分享成功率（点击后完成分享发送）>= 85%
- 社交分享带来的新用户下载转化 >= 3%（长期跟踪）

**Entry Points:**
- 从 03.3 Trend Chart 点击"看看你的消费雷达"CTA 进入
- 从 01.7 Dashboard 消费雷达入口（如有）进入
- 从社交分享链接打开 APP 后深度链接进入（Phase 2）

**Exit Points:**
- 点击"分享"生成卡片，发到好友/群聊 -> 场景完成 ✓
- 返回（左上角 / 安卓返回键）-> 03.3 Trend Chart
- 通过底部 Tab 导航到其他页面

## Content & Language Strategy

**叙事框架：「你的消费画像」**

03.1-03.3 是分析（数字、趋势、对比），03.4 是感性表达（画像、标签、趣味洞察）。雷达图把冰冷的消费数据变成一张"自画像"，消费标签把数据变成社交货币。小丽不会截图一个饼图发群，但她会截图"外卖达人"这个标签。

| 区域 | 语气 | 核心信息 |
|------|------|---------|
| 雷达图 | 视觉优先，一图胜千言 | 六维形状 = 你的消费DNA |
| 消费标签 | 趣味自嘲，社交友好 | "外卖达人"比"餐饮消费占比63%"有趣100倍 |
| 维度详情 | 简洁有料，一句话特征 | "外卖占68%，最爱瑞幸" -- 具体、有画面 |
| AI 趣味洞察 | 夸张有趣，可传播 | "绕地球0.3圈" -- 数字转化为画面感 |
| 月份对比 | 轻量客观 | 雷达形状变化一目了然，不做价值判断 |
| 分享区 | 友好邀请 | "分享给好友" -- 自然社交行为 |

**关键策略：**
- **审美第一** -- 小丽是"3秒判断好不好看"的用户（L4-，13/15）。雷达图、分享卡片、整个页面的视觉设计必须达到"值得截图"的标准
- **趣味>精确** -- 洞察文案追求"可分享性"而非"精确性"。"绕地球0.3圈"是近似值没关系，重要的是有画面感
- **隐私友好分享** -- 分享卡片默认不含具体金额。用户愿意分享"外卖达人"标签，但可能不想让朋友知道自己具体月花多少
- **真相镜子，不是法官** -- 消费标签全部正面/中性/自嘲。没有"月光族""剁手党"这种负面标签。"外卖达人"是陈述，不是批评

**语言：** 界面简体中文，代码/后端英文

## Design Decisions

1. **雷达图（Spider Chart）而非饼图或条形图** -- 雷达图的"形状"是最直觉的消费画像。每个人的雷达形状不同，天然具有社交分享性。饼图展示占比但缺乏"画像感"，条形图太普通
2. **六维而非更多** -- 六维是视觉清晰度和信息密度的平衡点。四维太少缺乏差异化，八维以上在手机屏幕上太密集。固定六个核心消费维度
3. **消费标签是核心社交货币** -- "外卖达人"四个字比整个雷达图更有传播力。标签设计遵循：正面/中性/自嘲，绝不负面
4. **分享卡片不含金额** -- 隐私第一。小丽愿意秀"外卖达人"身份，但不愿意让所有人知道她月花多少。雷达形状展示消费结构（占比），不暴露绝对值
5. **分享卡片 9:16 竖屏** -- 01.8 大强的报表卡片是 3:4（微信聊天场景），小丽的雷达卡片是 9:16（朋友圈/小红书场景）。不同角色，不同社交场景，不同卡片比例
6. **朋友圈和小红书渠道** -- 01.8 只有微信和保存。03.4 增加朋友圈和小红书，因为小丽的社交场景更广。她的分享不只是"给某个人看"，而是"晒给所有人看"
7. **分享成功文案"坐等评论"** -- 01.8 大强的是"有据可说"（呼应家庭沟通），03.4 的是"坐等评论"（呼应社交互动期待）。不同场景定制化的情感反馈，措辞保持性别中立
8. **雷达分数基于占比+频次** -- 纯金额占比会让房租/房贷等固定支出扭曲雷达形状。加入频次权重后，高频小额消费（奶茶、外卖）的维度分更准确地反映消费"习惯"
9. **月份对比用双雷达并排** -- 比动画切换（前后叠加）更直观。用户可以同时看到两个月的雷达形状差异，不需要在脑中记忆前一个形状
10. **AI 洞察模板化+具体化** -- "绕地球0.3圈"这类夸张表述需要具体数据支撑（杯数 * 杯口直径 = 距离）。后端提供计算，前端套用模板。保证每条洞察都有"可传播性"
11. **纸飞机而非对勾** -- 01.8 大强分享成功用对勾（庄重、任务完成），03.4 小丽用纸飞机（活泼、社交飞出）。角色气质不同，动效也不同

## Technical Notes

- **雷达图组件：** 使用 `react-native-svg` 手绘六边形蛛网 + 数据区域填充。推荐封装为通用 `<RadarChart>` 组件，参数：维度数组、分数数组、颜色、大小
- **雷达图动画：** 使用 `react-native-reanimated` 的 `SharedValue` 控制从中心到目标坐标的 spring 动画
- **消费标签生成：** 后端计算各维度占比 + 子项特征（外卖占比、品牌频次），匹配标签规则表，返回最匹配的标签
- **AI 趣味洞察：** 后端预计算统计指标（饮品杯数估算、打车里程估算、时段分布），AI 生成自然语言描述。模板 + 变量填充，不完全依赖 LLM
- **分享卡片生成：** 使用 `react-native-view-shot` 截图专用 View。卡片 View 独立渲染，背景渐变用 `expo-linear-gradient`
- **微信朋友圈分享：** 微信 SDK `shareToTimeline` 接口，类型为图片分享
- **雷达分数计算：** 每维度分数 = (该维度金额占比 * 0.6 + 该维度笔数占比 * 0.4) * 100。固定支出（房贷等）降权处理（乘以 0.3 系数）
- **月份对比雷达：** 两个 `<RadarChart>` 实例并排，数据独立加载。切换月份时两个图同步更新
- **分类-颜色映射：** 复用全局映射表（与 01.8 和 03.3 一致）
- **分享卡片缓存：** 生成后缓存到本地（按月份 + 用户 ID 为 key），同月再次分享时跳过生成步骤
- **深度链接：** 分享卡片二维码链接格式 `moneytracker://radar?month=2026-03`，未安装用户跳转到应用商店
