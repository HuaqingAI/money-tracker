## Overview

**Page Purpose:** 展示某个身份（"老板小美"）的独立损益表——这是整个场景的**成功时刻**。小美第一次看到副业的真实净利润（¥2,847）和时薪（¥23/小时），从"模糊的期待"变成"清晰的数字"。

**Design Concept: 真相镜子** -- 延续 03.x "消费真相镜子"的设计哲学。这不是审判（"你亏了"），而是呈现事实（"你的净利润是 2847"）。即便数字不理想，清晰本身就是价值——小美终于可以做理性决策了。

**User Situation:** 小美刚在 05.1 创建了"老板小美"身份并确认了归属规则，被引导到这里看结果。或者，日常使用中从 05.1 身份列表点击卡片进入。心态是"给我看数字"——高度期待又轻微紧张。

**Success Criteria:**
- 页面到达率 >= 95%（从 05.1 完成到达此页的比例）
- 首屏停留时间 >= 8s（说明用户在认真看数字）
- 成本明细展开率 >= 60%（用户关注成本结构）
- 时薪查看率 >= 70%（滚动到时薪区域的用户比例）
- 月度切换率 >= 30%（用户查看其他月份的比例）

**Entry Points:**
- 从 05.1 规则确认完成后自动进入（场景主路径）
- 从 05.1 Identity List 点击身份卡片进入（日常使用）
- 从 Dashboard 身份切换器进入（快捷入口）

**Exit Points:**
- 返回 -> 05.1 Identity List 或 Dashboard（视入口而定）
- "调整归属规则" -> 05.1 规则管理
- 分享损益摘要 -> 系统分享面板（Phase 2）

## User Mental State

| 维度 | 状态 |
|------|------|
| 触发事件 | 刚确认完归属规则，或从身份列表点进来 |
| 情绪基调 | 高度期待 + 紧张（"终于要看到那个数字了"） |
| 希望 | "希望是赚了的，哪怕少也行"、"给我一个清晰的全貌" |
| 恐惧 | "万一是亏的怎么办？"（M1- 在此刻达到顶峰） |
| 内心问题 | "净利润多少？""到底赚了还是亏了？""我的时间值多少钱？" |
| 耐心程度 | 高 -- 这是她一直想看到的页面，会仔细阅读每一个数字 |
| 关键信号 | 看到净利润正数时会松一口气 + 开始计算时薪时会深度思考 |

**心态转变设计目标：**

```
进入时              过程中                结束时
紧张期待  -->  数字释然  -->  理性决策
"赚了还是亏了？"  "原来赚了2847"   "投流ROI太差，下月调整"
```

## Content & Language Strategy

**核心叙事："一个清晰的数字"**

小美来这里只为一件事：看到那个数字。设计的核心是让数字本身成为故事的主角。

| 区域 | 叙事策略 | 核心信息 |
|------|---------|---------|
| Hero | 揭晓时刻 | 40pt 大号数字 + 数字滚动动画——仪式感等于"重要" |
| 环比变化 | 趋势暗示 | "比上月多 ¥340"——正面方向感 |
| 利润率 | 商业视角 | 31.6%——"每赚100块留下31块6"，不只是一个绝对数字 |
| 成本明细 | 隐性成本可视化 | M3+ 的直接回应。投流占 32% 比"投流花了 2000"更有冲击力 |
| AI 洞察 | 可执行建议 | 不是"你花了很多钱"而是"ROI 2.7 低于行业平均 4.0"——给出参照系和方向 |
| 时薪 | 人生决策数据 | ¥23/小时 vs ¥51/小时——这是 M4+ 的终极数据。不评判，只呈现 |
| 全职预告 | 目标驱动留存 | "连续3个月超过主业"——明确的里程碑，制造"继续用下去"的动力 |

**情感设计：渐进式揭示**
- 先看净利润（正面的"赚了"）
- 再看成本结构（"原来花了这么多在投流上"）
- 最后看时薪（"我的时间值多少"——可能是最扎心但最有价值的数字）

**亏损场景的情感保护：**
- 净利润为负时用橙色而非红色
- 文案不用"亏损"用"净支出超过收入"
- AI 洞察侧重"优化方向"而非"问题诊断"
- 例如："收入 ¥3,200，成本 ¥4,800。如果将投流预算控制在 ¥1,000 以内，下月有望转正" 而非 "本月亏损 ¥1,600"

**语言：** 界面简体中文，代码/后端英文

## Design Decisions

1. **净利润 Hero Number 40pt** -- 这是整个场景的"揭晓时刻"。数字必须够大、够醒目，配合从 0 滚动的动画，制造仪式感。小美盯着这个数字看的时候，就是她从"模糊期待"变成"清晰认知"的瞬间
2. **利润率而非纯利润** -- "31.6%"比"2847"更有商业判断价值。小美不只是个记账用户，她是个小老板——给她老板级的数据视角
3. **成本明细降序排列** -- 最大的成本在最前面。小美一眼看到"投流占 32%"——这个信息足以驱动她的下月决策
4. **AI 洞察带参照系** -- "ROI 2.7（行业平均 4.0+）"比"ROI 2.7"有用得多。参照系让数字有了判断基准
5. **时薪对比是核心** -- 副业时薪 ¥23 vs 主业时薪 ¥51。这个对比回答了小美心里最深的问题："副业值不值得做？"不评判，只呈现数字
6. **时间投入由 AI 估算+用户可调** -- AI 根据交易时间戳推算副业时间（晚间+周末），但允许用户手动修正。精度不完美，但"大致准确"比"没有"有价值得多
7. **亏损用橙色不用红色** -- 延续全 APP "真相镜子而非法官"的设计原则。亏损是商业信息，不是错误提示
8. **全职可行性预告** -- "3/3 个月达标"的进度条制造清晰的目标感。小美会想"再努力两个月就能解锁深度分析了"——这是天然的留存引擎
9. **月份切换器** -- 不用左右滑动切换（容易误触），用下拉选择器（精确选择）。副业用户经常需要看特定月份（如"上个月投流效果如何"）
10. **成本堆叠条形图** -- 比饼图更直觉地展示"大头在哪"。水平条在手机上更紧凑，不占纵向空间
11. **趋势图三线叠加** -- 净利润（实线）+ 收入（虚线）+ 支出（虚线）。小美可以看到"收入在涨但成本涨得更快"——趋势背后的原因

## Technical Notes

- **P&L 数据 API：** `GET /api/identities/{identity_id}/pnl?month={YYYY-MM}` 返回 `{ net_profit, income_total, expense_total, income_breakdown: [...], cost_breakdown: [...], hours_estimated, hourly_rate, trend: [...], attribution: { assigned, pending } }`
- **成本分类引擎：** 后端将副业支出按规则类别自动分组（投流/样品/物流/佣金/售后等），前端直接渲染
- **时薪计算：** `hourly_rate = net_profit / hours`。hours 来源优先级：用户手动输入 > AI 估算
- **AI 估算时间：** 分析该身份交易的时间戳分布，推算"副业活跃时段"（如 20:00-24:00 + 周末），累加活跃小时数。精度约 ±20%
- **环比计算：** 当前月 net_profit - 上月 net_profit = diff。无上月数据则不展示
- **利润率计算：** margin = net_profit / income_total * 100。income_total = 0 时不展示利润率
- **趋势数据：** `GET /api/identities/{identity_id}/trend?months=6` 返回最近 6 个月的月度汇总
- **AI 洞察引擎：** 基于成本结构 + 行业基准（预配置的电商/自由职业/投资基准数据）生成建议。初期为模板匹配，后续升级为 ML 生成
- **图表库：** React Native Chart Kit 或 Victory Native，折线图 + 柱状图 + 堆叠条形图
- **缓存策略：** P&L 数据按月缓存，当月数据 5 分钟 TTL（新交易可能影响），历史月份数据 1 小时 TTL
