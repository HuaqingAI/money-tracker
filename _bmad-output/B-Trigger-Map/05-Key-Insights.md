## The Flywheel: Priorities That Drive Everything

**⭐ Priority #1: THE ENGINE -- 零记账体验**
- 三个群体的最高分驱动力全部指向同一件事：**不要让我手动记**
- 大强 D2+ (15/15)、小丽 L1- (15/15)、小美 M4- (13/15)
- 如果这个不成立，其他一切都不存在

**🚀 Priority #2: GROWTH -- 真相呈现**
- 三个群体都渴望看到真实数字，但"真相"的含义不同
- 大强要家庭消费归属（"谁花的、为谁花的"）
- 小丽要消费模式（"钱被什么偷走了"）
- 小美要副业利润（"到底赚了没有"）

**🌟 Priority #3: SUSTAINABILITY -- 差异化功能**
- 共享家庭账本（大强）、人情账管理（大强）、多身份核算（小美）
- 这三个功能在市场上几乎没有竞品做
- 它们是付费转化的核心驱动力

## Primary Development Focus

### 1. Zero-Input as Identity, Not Feature
零输入不是一个功能 -- 它是产品的身份。一切设计决策都应该问："这会不会让用户多动一下手？"如果会，要么自动化它，要么砍掉它。

### 2. Truth Mirror, Not Judge
产品是一面镜子，不是一个法官。展示"你这个月奶茶花了800"，不展示"你超支了！"。正向激励 > 负面审判。小丽的 L2- 和大强的 D2- 都在说同一件事：不要让我因为看到数据而感到更糟。

### 3. Shared Facts Dissolve Conflicts
大强最高分的正面驱动力（D1+, 14/15）揭示了一个深层洞察：记账的价值不只是个人财务管理，而是**关系修复**。共享的、自动生成的数据消除了"说不清"的根因。

### 4. Identity-Aware Accounting
小美的核心需求（M2+, 14/15）指向一个传统记账APP完全忽视的维度：同一个人可以有多个财务身份。这不是"多账本"那么简单 -- 而是消费的**归属**可以根据身份改变。

### 5. Gift Economy as Blue Ocean
人情账管理（大强 D3+/D4-）在竞品中几乎不存在。93%的年轻人感受随礼负担，但没有任何工具帮他们管理。这是最纯粹的蓝海功能。

## Critical Success Factors

1. **AI捕获覆盖率 >= 80%（Android）** -- 低于这个阈值，"零记账"就是空话
2. **首次体验 <= 3分钟出结果** -- 小丽的放弃阈值定义了产品的新手体验上限
3. **UI设计年轻化** -- 小丽的 L4-（13/15）说明界面审美是硬门槛，不是锦上添花
4. **数据安全可感知** -- 大强的 D5-（11/15）不高但是信任门槛，不满足则一切归零

## Design Implications by Section

### Onboarding
- **Must:** 导入账单 → 看到自动分类报表，全程 < 3分钟
- **Must:** 零手动输入，不要求选分类、写备注
- **Should:** 展示"你的记账覆盖率已达 X%"（建立信任）
- **Avoid:** 复杂注册流程、功能教程、权限弹窗轰炸

### Dashboard
- **Must:** 打开就是消费概览，不需要任何操作
- **Must:** 正向情绪设计，不用红色警报
- **Should:** 高亮有趣的消费模式（"你这个月最爱的品牌是...""咖啡消费趋势"）
- **Avoid:** Excel式数字堆砌，"你超支了"式的审判

### Reporting
- **Must:** 自动生成月度报表，可一键分享
- **Should:** 设计精美的可视化（雷达图、趋势线、分类饼图）
- **Should:** 支持分享到微信/小红书的卡片格式
- **Avoid:** 纯文字报表，无法分享的格式

### Family Features (Phase 2)
- **Must:** 共享账本双方看到同一份数据
- **Should:** 受益人维度标注（"为谁花的"）
- **Should:** 各自隐私消费不共享
- **Avoid:** 变成"监控工具"的感觉

### Gift Accounting (Phase 2)
- **Must:** 按人管理随礼历史
- **Should:** 智能建议回礼区间
- **Should:** 季节性预警（"下个月有3场婚礼，预计约3000"）
- **Avoid:** 过于复杂的人情关系图谱（MVP保持简单）

### Multi-Identity (Phase 2)
- **Must:** 一键切换消费归属身份
- **Should:** AI学习归属模式（"小红书平台消费 → 自动归副业"）
- **Should:** 各身份独立损益表
- **Avoid:** 过度简化成"多账本"（身份 ≠ 账本）

## Design Focus Statement

Money Tracker 的设计核心是**把财务真相变成零负担的自然体验**。

**Primary Design Target:** 大强 Danny（如果他能用，所有人都能用）

**Must Address:**
- 零输入零精力（三人共识15/15）
- 自动消费真相（三人共识14/15）
- 共同事实基础（大强核心需求14/15）

**Should Address:**
- 年轻化UI（小丽门槛13/15）
- 消费模式发现（小丽+小美11-13/15）
- 数据安全感知（大强信任门槛11/15）

## Development Phases (Aligned with Flywheel)

### Phase 1: MVP -- 验证零记账 (THE ENGINE)
- 账单导入（支付宝/微信）
- AI自动分类
- 基础报表（月度概览、分类占比）
- **验证：** 大强能不动手看到报表？小丽3分钟内出结果？

### Phase 2: 差异化 -- 多维理解
- 共享家庭账本（大强）
- 受益人标注（大强）
- 人情账管理（大强）
- 多身份核算（小美）
- 消费习惯雷达（小丽）
- **验证：** 差异化功能使用率 >= 30%？付费转化 >= 5%？

### Phase 3: AI管家 -- 智能层
- AI对话
- 主动洞察推送
- 优化建议
- 消费模拟
- 同类对比
- **验证：** 用户NPS提升？留存率进一步提高？
